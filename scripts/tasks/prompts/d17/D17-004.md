# Task D17-004: Add early bird rate lock columns to subscriptions table

## What to Build

Modify the existing subscriptions table in `packages/shared/src/schemas/db/platform.schema.ts` to add two new columns for early bird rate lock tracking.

**Add the following columns to the `subscriptions` table definition, after the `deletionScheduledAt` column:**

```typescript
// --- Early Bird Rate Lock (Batch 2: Pricing Lifecycle, spec B2-2) ---
// Set to created_at + 12 months on early bird checkout completion.
// Null for non-early-bird plans.
earlyBirdLockedUntil: timestamp('early_bird_locked_until', {
  withTimezone: true,
}),

// Tracks whether the 30-day expiry warning notification has been sent.
// Prevents duplicate notification sends in the scheduled job.
earlyBirdExpiryNotified: boolean('early_bird_expiry_notified')
  .notNull()
  .default(false),
```

**Add an index for the scheduled job query** (finding expiring early bird subscriptions):

Add to the table's index array:
```typescript
index('subscriptions_early_bird_locked_until_idx').on(table.earlyBirdLockedUntil),
```

**Schema rules (from spec B2-2):**
- `early_bird_locked_until` is TIMESTAMPTZ, nullable. Null means this is not an early bird subscription.
- `early_bird_expiry_notified` is BOOLEAN, default false. Set to true after the 30-day warning notification is emitted.
- The combination of these columns is used by the `checkEarlyBirdExpiry()` scheduled job (D17-012) to find subscriptions approaching expiry.

**Do NOT modify** any other columns or tables. Only add the two new columns and the index.

**Update inferred types** — the existing `InsertSubscription` and `SelectSubscription` types will automatically include the new columns since they use `$inferInsert` / `$inferSelect`.

## Critical Security Rules

- These columns contain no PHI — they are billing lifecycle metadata.
- The `early_bird_locked_until` timestamp must use `withTimezone: true` (TIMESTAMPTZ) to ensure correct UTC handling across time zones.
- The `early_bird_expiry_notified` flag prevents notification spam — the scheduled job must check this before emitting.

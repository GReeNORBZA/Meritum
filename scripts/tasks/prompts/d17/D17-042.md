# Task D17-042: Security — Early bird expiry transitions and orphaned subscription prevention

## What to Build

Create `apps/api/test/security/platform/platform.earlybird-expiry.security.ts` to verify that the early bird expiry transition process works correctly and does not leave orphaned subscriptions, double-billing scenarios, or billing gaps.

**Security test cases:**

```typescript
import { describe, it, expect, beforeAll, afterAll } from 'vitest';

describe('Security: Early Bird Expiry Transitions', () => {

  describe('No orphaned subscriptions (individual path)', () => {
    it('after transition from EARLY_BIRD_MONTHLY to STANDARD_MONTHLY: only one active subscription exists');
    it('after transition from EARLY_BIRD_ANNUAL to STANDARD_ANNUAL: only one active subscription exists');
    it('the old early bird Stripe subscription is cancelled in Stripe');
    it('the new standard Stripe subscription is active in Stripe');
    it('no period gap: new subscription starts from where early bird left off');
  });

  describe('No orphaned subscriptions (practice path)', () => {
    it('after transition to PRACTICE_CONSOLIDATED: individual Stripe subscription is cancelled');
    it('after transition: practice Stripe subscription quantity is incremented by exactly 1');
    it('no double billing: physician is NOT billed individually AND through practice simultaneously');
    it('practice membership billing_mode is PRACTICE_CONSOLIDATED after transition');
    it('previous INDIVIDUAL_EARLY_BIRD billing_mode no longer exists for this member');
  });

  describe('Idempotency — no duplicate transitions', () => {
    it('running checkEarlyBirdExpiry twice does not cancel subscription twice');
    it('running checkEarlyBirdExpiry twice does not increment practice quantity twice');
    it('running checkEarlyBirdExpiry twice does not send duplicate notifications');
    it('already-transitioned subscription is skipped on subsequent runs');
  });

  describe('Failure recovery', () => {
    it('if Stripe cancellation fails: local subscription record is NOT updated (transaction rollback)');
    it('if Stripe price update fails: subscription plan is NOT changed locally');
    it('if practice quantity increment fails: membership billing_mode is NOT changed');
    it('partial failure: physician still has a valid active subscription after any failure');
  });

  describe('Concurrent transitions in same practice', () => {
    it('two physicians in same practice expiring simultaneously: each gets quantity +1 (total +2)');
    it('concurrent transitions do not cause quantity miscalculation');
    it('all notifications are sent for each physician independently');
  });

  describe('Edge cases', () => {
    it('physician removed from practice before early bird expires: treated as individual transition (Path B)');
    it('practice dissolved before early bird expires: physician gets individual standard subscription');
    it('physician with early bird locked_until = null is ignored by the scheduled job');
    it('physician with cancelled early bird subscription is ignored (already cancelled)');
    it('physician with suspended early bird subscription is still processed at expiry');
  });

  describe('Billing continuity', () => {
    it('no billing gap between early bird cancellation and practice consolidated start');
    it('no billing gap between early bird and standard transition');
    it('physician retains access throughout the transition (no feature lockout)');
    it('subscription status remains ACTIVE during price change transition (Path B)');
  });

  describe('Notification correctness', () => {
    it('EARLY_BIRD_EXPIRED notification includes new rate information');
    it('PRACTICE_MEMBER_TRANSITIONED notification to admin does NOT include billing amounts');
    it('notifications are sent AFTER successful transition (not before)');
    it('failed transitions do not generate false notifications');
  });
});
```

**Test setup requirements:**
- Create test practices with various member configurations (all early bird, mixed, none early bird)
- Set up subscriptions with `early_bird_locked_until` in the past (to trigger transitions)
- Mock Stripe API to verify cancellation, price update, and quantity changes
- Mock notification system to capture and verify emitted notifications
- Use transactions to test failure recovery scenarios

**What to verify:**
1. After any early bird expiry, the physician has exactly ONE active billing relationship (either individual Stripe subscription OR practice consolidated — never both, never zero)
2. Practice Stripe quantity matches the count of `PRACTICE_CONSOLIDATED` members at all times
3. The scheduled job is idempotent — safe to run multiple times
4. Stripe failures result in database rollback (no inconsistent state)
5. Notifications are accurate and privacy-safe

## Critical Security Rules

- Orphaned subscriptions = double billing or lost revenue. This is a financial integrity test.
- The idempotency tests are critical — cron jobs can run multiple times due to infrastructure issues.
- Failure recovery tests ensure that Stripe and database remain consistent even when external APIs fail.
- Practice admin notifications must NEVER contain individual physician billing data (spec B1-11).

# Task D17-013: Service — Early bird + clinic interaction (acceptInvitation billing_mode)

## What to Build

Modify the `acceptInvitation()` function in the practice service layer (`apps/api/src/domains/platform/platform.service.ts` or `apps/api/src/domains/platform/practice.service.ts` — whichever was created in Batch 1) to correctly determine `billing_mode` based on the accepting physician's early bird status.

Also modify `createPractice()` to handle founding members who are on early bird plans.

**acceptInvitation() modification (spec B2-4):**

The function already creates a practice membership when a physician accepts an invitation. The modification is to the `billing_mode` determination:

```typescript
// In acceptInvitation(), after validating the invitation:

// Determine billing_mode based on early bird status
const activeSubscription = await repository.getActiveSubscription(physicianUserId);

let billingMode: string;

if (activeSubscription && isEarlyBirdRate(activeSubscription.plan) && activeSubscription.earlyBirdLockedUntil && activeSubscription.earlyBirdLockedUntil > new Date()) {
  // Physician has an active early bird subscription with unexpired rate lock
  // They keep their individual early bird subscription
  // Practice Stripe quantity is NOT incremented
  billingMode = 'INDIVIDUAL_EARLY_BIRD';
} else {
  // Physician transitions to practice consolidated billing
  // Cancel their individual subscription (if any)
  // Increment practice Stripe subscription quantity
  billingMode = 'PRACTICE_CONSOLIDATED';

  if (activeSubscription) {
    await stripe.subscriptions.cancel(activeSubscription.stripeSubscriptionId);
    await repository.updateSubscription(activeSubscription.subscriptionId, {
      status: 'CANCELLED',
      cancelledAt: new Date(),
    });
  }

  // Increment practice Stripe quantity
  await incrementPracticeStripeQuantity(practiceId);
}

// Create membership with determined billing_mode
await repository.createPracticeMembership({
  practiceId,
  physicianUserId,
  billingMode,
  joinedAt: new Date(),
  isActive: true,
});
```

**Key business rules (from spec B2-4):**
- Early bird and clinic tier DO NOT stack — a physician on early bird keeps $199, not $251.10
- Each early bird signup counts against the 100-spot cap independently
- Early bird physicians joining a practice are full members, count toward the 5-physician minimum
- They remain on individual billing until their rate lock expires (handled by D17-012)
- The practice Stripe subscription quantity only counts `PRACTICE_CONSOLIDATED` members
- A practice could have total_members = 7 but Stripe quantity = 4 (if 3 are early bird)

## Critical Security Rules

- The billing_mode determination must be server-side — a client cannot choose their billing_mode.
- The early bird lock check must compare `earlyBirdLockedUntil > new Date()` — expired locks are treated as non-early-bird.
- Stripe operations (cancel individual, increment practice) must be atomic with the database updates — use a transaction.
- Audit log all billing_mode determinations for financial accountability.

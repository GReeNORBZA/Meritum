# Task D17-014: Service — Proactive transition notifications for practice admins

## What to Build

Add proactive notification logic to the early bird expiry workflow in `apps/api/src/domains/platform/platform.service.ts`. This extends the `checkEarlyBirdExpiry()` job from D17-012 with practice-admin-facing notifications.

**Notification 1: First physician approaching expiry (spec B2-5)**

When the `checkEarlyBirdExpiry()` job finds a physician in a practice whose early bird is expiring within 30 days, AND this is the first member of that practice to approach expiry:

```typescript
// After emitting EARLY_BIRD_EXPIRING to the physician (existing from D17-012):

// Check if this is the first practice member approaching early bird expiry
const practiceMembership = await repository.getActivePracticeMembership(subscription.providerId);
if (practiceMembership) {
  const practice = await repository.getPractice(practiceMembership.practiceId);
  const earlyBirdMembers = await repository.getEarlyBirdMembersInPractice(practiceMembership.practiceId);

  // Check if any other member has already been notified about expiry
  const anyPreviouslyNotified = earlyBirdMembers.some(
    m => m.physicianUserId !== subscription.providerId &&
         m.earlyBirdExpiryNotified === true
  );

  if (!anyPreviouslyNotified) {
    // This is the FIRST physician approaching expiry in this practice
    await emitNotification({
      type: 'PRACTICE_EARLY_BIRD_TRANSITION_STARTING',
      recipientUserId: practice.adminUserId,
      data: {
        practiceId: practice.practiceId,
        practiceName: practice.name,
        // Do NOT include physician name or billing details
        message: 'A physician in your practice has an early bird rate expiring soon. Their billing will automatically transition to practice consolidated billing.',
      },
    });
  }
}
```

**Notification 2: All physicians post-early-bird — suggest clinic tier (spec B2-5)**

After each early bird expiry transition (in the Phase 2 Path A handler from D17-012), check if all practice members are now post-early-bird:

```typescript
// After transitioning a member from INDIVIDUAL_EARLY_BIRD to PRACTICE_CONSOLIDATED:

const remainingEarlyBird = await repository.getEarlyBirdMembersInPractice(practiceId);
if (remainingEarlyBird.length === 0) {
  // All physicians on this practice are now post-early-bird
  // All are on PRACTICE_CONSOLIDATED billing
  await emitNotification({
    type: 'PRACTICE_ALL_MEMBERS_POST_EARLY_BIRD',
    recipientUserId: practice.adminUserId,
    data: {
      practiceId: practice.practiceId,
      practiceName: practice.name,
      totalMembers: activeMemberCount,
      message: 'All physicians in your practice have transitioned from early bird rates. Your practice is now fully on clinic tier consolidated billing.',
    },
  });
}
```

**Notification types to add to the notification event types (if a notification constants file exists):**
- `PRACTICE_EARLY_BIRD_TRANSITION_STARTING` — sent to practice admin when first member approaches expiry
- `PRACTICE_ALL_MEMBERS_POST_EARLY_BIRD` — sent to practice admin when all members are post-early-bird

## Critical Security Rules

- Practice admin notifications must NEVER include individual physician billing amounts, plan details, or subscription IDs.
- The `PRACTICE_EARLY_BIRD_TRANSITION_STARTING` notification refers generically to "a physician" — not by name, per the privacy rules in spec B1-11.
- The `PRACTICE_ALL_MEMBERS_POST_EARLY_BIRD` notification only includes aggregate data (total members), not per-member details.
- All notifications are audit logged.

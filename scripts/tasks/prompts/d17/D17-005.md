# Task D17-005: Generate and apply migration for early bird rate lock columns

## What to Build

Generate a Drizzle database migration for the schema changes made in D17-004 (adding `early_bird_locked_until` and `early_bird_expiry_notified` columns to the `subscriptions` table).

**Steps:**

1. Run `pnpm drizzle-kit generate` from the `apps/api` directory to generate the migration SQL.
2. Review the generated migration to ensure it contains:
   - `ALTER TABLE subscriptions ADD COLUMN early_bird_locked_until TIMESTAMPTZ;`
   - `ALTER TABLE subscriptions ADD COLUMN early_bird_expiry_notified BOOLEAN NOT NULL DEFAULT false;`
   - `CREATE INDEX subscriptions_early_bird_locked_until_idx ON subscriptions (early_bird_locked_until);`
3. Run `pnpm drizzle-kit migrate` to apply the migration.

**Important:** The migration should ONLY contain the two new columns and the index. If Drizzle generates additional changes (from other pending schema modifications), verify they are expected and do not drop or alter existing columns.

## Prerequisites

This task depends on: D17-004 (schema changes must be in platform.schema.ts before generating migration)

## Tests to Write

No test file for this task — the migration itself is the verification.

## Critical Security Rules

- Never drop existing columns or tables in a migration.
- The migration must be additive only (ALTER TABLE ADD COLUMN, CREATE INDEX).
- Verify that no data loss can occur — both new columns have safe defaults (NULL and false).

## Run After Completion

```bash
cd apps/api && pnpm drizzle-kit generate 2>&1 && pnpm drizzle-kit migrate 2>&1
```

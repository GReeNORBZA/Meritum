# Task D18-010: Repository: referral code CRUD operations

## What to Build

Add referral code repository functions to the platform domain repository layer. Create a new file `apps/api/src/domains/platform/referral.repository.ts` (or add to existing `platform.repository.ts` — follow the pattern established in the project).

**Functions to implement:**

### createReferralCode(referrerUserId: string, code: string)
- Insert a new row into `referral_codes` with the given `referrer_user_id` and `code`
- `is_active` defaults to `true` (from schema default)
- Returns the created `SelectReferralCode` record
- Uses Drizzle ORM insert

### findReferralCodeByCode(code: string)
- Select from `referral_codes` where `code = $1`
- Returns `SelectReferralCode | null`
- Used during code redemption to look up the code and its owner

### findReferralCodeByUserId(userId: string)
- Select from `referral_codes` where `referrer_user_id = $1` and `is_active = true`
- Returns `SelectReferralCode | null`
- Each physician has at most one active referral code
- Used by GET /referrals/my-code

### deactivateReferralCode(referralCodeId: string)
- Update `referral_codes` set `is_active = false` where `referral_code_id = $1`
- Returns the updated record
- Used when a physician's subscription is cancelled (their code should no longer be usable)

**Dependency injection pattern:** Follow the same pattern as other repositories in the project. Accept a Drizzle `db` instance (or use the shared database connection from the project's DI setup).

**Import the schema** from `packages/shared/src/schemas/db/platform.schema.ts` — use the redesigned `referralCodes` table (from D18-001).

This task depends on the following completed tasks: D18-001, D18-004

## Tests to Write

Add tests to `apps/api/src/domains/platform/platform.test.ts` (or a dedicated `referral.test.ts` file within the same directory):

```typescript
describe('ReferralCodeRepository', () => {
  it('createReferralCode inserts a new code and returns the record');
  it('createReferralCode defaults is_active to true');
  it('findReferralCodeByCode returns the code record for a valid code');
  it('findReferralCodeByCode returns null for non-existent code');
  it('findReferralCodeByUserId returns the active code for a user');
  it('findReferralCodeByUserId returns null if no active code exists');
  it('findReferralCodeByUserId ignores deactivated codes');
  it('deactivateReferralCode sets is_active to false');
  it('deactivateReferralCode returns the updated record');
});
```

## Critical Security Rules

- All queries must be parameterized (Drizzle ORM handles this, but verify no raw SQL interpolation).
- `referrer_user_id` must always come from an authenticated session, never from request body.
- Code lookups are case-sensitive — codes are always uppercase alphanumeric.

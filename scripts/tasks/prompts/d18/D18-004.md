# Task D18-004: Zod validation schemas for referral endpoints + database migration

## What to Build

### Part 1: Zod Schemas

Add referral-related Zod validation schemas to the appropriate shared schema file (likely `packages/shared/src/schemas/platform.schema.ts` or a new `packages/shared/src/schemas/referral.schema.ts` — follow the existing pattern in the project).

**Schemas to create:**

```typescript
// POST /api/v1/signup/referral-code — public validation
export const validateReferralCodeSchema = z.object({
  body: z.object({
    code: z.string().length(8).regex(/^[A-Z0-9]{8}$/, 'Invalid referral code format'),
  }),
});

// POST /api/v1/referrals/:redemptionId/apply — clinic physician credit choice
export const applyReferralCreditSchema = z.object({
  params: z.object({
    redemptionId: z.string().uuid(),
  }),
  body: z.object({
    target: z.enum(['PRACTICE_INVOICE', 'INDIVIDUAL_BANK']),
  }),
});

// GET /api/v1/referrals/my-code — no body, auth only
export const getMyReferralCodeSchema = z.object({});

// GET /api/v1/referrals/my-credits — optional query params
export const getMyReferralCreditsSchema = z.object({
  query: z.object({
    status: z.enum(['PENDING', 'QUALIFIED', 'CREDITED', 'EXPIRED']).optional(),
    anniversaryYear: z.coerce.number().int().min(1).optional(),
  }).optional(),
});
```

**Export all schemas** from the shared package barrel file.

### Part 2: Database Migration

Generate and apply the database migration for the redesigned referral tables (from D18-001 and D18-002).

Run:
```bash
cd apps/api && pnpm drizzle-kit generate
```

This will detect the schema changes (replaced `referral_codes` and `referral_redemptions` columns) and generate a migration file.

**Review the generated migration SQL** to confirm it:
1. Drops the old columns: `redemption_count`, `max_redemptions`, `physician_user_id` (from referral_codes), `credit_amount_cad`, `credit_applied` (from referral_redemptions)
2. Adds the new columns to both tables
3. Renames `physician_user_id` → `referrer_user_id` (or drops and recreates if Drizzle doesn't support rename)
4. Creates the new indexes

**If drizzle-kit generates a destructive migration** (DROP TABLE + CREATE TABLE instead of ALTER TABLE), that is acceptable for this case since the referral tables contain no production data yet (they were MVP scaffolding).

Then apply:
```bash
cd apps/api && pnpm drizzle-kit migrate
```

## Critical Security Rules

- The `code` field regex enforces uppercase alphanumeric only — no special characters.
- `redemptionId` must be validated as UUID to prevent injection.
- `target` is a strict enum — no arbitrary strings accepted.

# Task D18-002: Redesign referral_redemptions table

## What to Build

**REPLACE** the existing `referral_redemptions` table in `packages/shared/src/schemas/db/platform.schema.ts`. The old schema tracks a simple boolean `credit_applied` with a fixed $50 credit. The new schema implements a full status workflow with variable credit values per the Pricing Gap Closure Spec (B3-1).

**Delete the entire old table definition** and replace with the following columns:

| Column | Drizzle Definition | Constraints |
|--------|-------------------|-------------|
| redemption_id | `uuid('redemption_id').primaryKey().defaultRandom()` | PK |
| referral_code_id | `uuid('referral_code_id').notNull().references(() => referralCodes.referralCodeId)` | FK → referral_codes |
| referrer_user_id | `uuid('referrer_user_id').notNull().references(() => users.userId)` | FK → users (denormalized for queries) |
| referred_user_id | `uuid('referred_user_id').notNull().references(() => users.userId)` | FK → users |
| status | `varchar('status', { length: 20 }).notNull().default('PENDING')` | One of: PENDING, QUALIFIED, CREDITED, EXPIRED |
| credit_month_value_cad | `decimal('credit_month_value_cad', { precision: 10, scale: 2 })` | Nullable — set when QUALIFIED |
| credit_applied_to | `varchar('credit_applied_to', { length: 20 })` | Nullable — PRACTICE_INVOICE or INDIVIDUAL_BANK |
| credit_applied_at | `timestamp('credit_applied_at', { withTimezone: true })` | Nullable — when credit was applied |
| qualifying_event_at | `timestamp('qualifying_event_at', { withTimezone: true })` | Nullable — when referred completed one billing cycle |
| anniversary_year | `integer('anniversary_year').notNull()` | Referrer's signup anniversary year (for 3-per-year cap) |
| created_at | `timestamp('created_at', { withTimezone: true }).notNull().defaultNow()` | |

**Indexes:**
- `index('referral_redemptions_referral_code_id_idx').on(table.referralCodeId)`
- `index('referral_redemptions_referred_user_id_idx').on(table.referredUserId)`
- `index('referral_redemptions_referrer_user_id_idx').on(table.referrerUserId)`
- `index('referral_redemptions_status_idx').on(table.status)`
- `index('referral_redemptions_referrer_anniversary_idx').on(table.referrerUserId, table.anniversaryYear)`

**Remove the old MVP accommodation comment** about `$50 credit per referral`.

**Update the inferred types** at the bottom of the file:
- `InsertReferralRedemption` and `SelectReferralRedemption` must reflect the new schema

**Status workflow:**
- `PENDING` — code redeemed, waiting for referred user to complete one billing cycle
- `QUALIFIED` — referred user completed billing cycle, credit value calculated
- `CREDITED` — credit applied to referrer's invoice/account
- `EXPIRED` — referral did not qualify (3-per-year cap exceeded, or referred user never paid)

**Why `referrer_user_id` is denormalized:** The referrer can be looked up via `referral_code_id → referral_codes.referrer_user_id`, but having it directly on redemptions enables efficient queries for "count all qualified credits for referrer X in anniversary year Y" without a join.

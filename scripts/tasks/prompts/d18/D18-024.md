# Task D18-024: Service: referee incentive — first month free after early bird window closes

## What to Build

Add referee (referred physician) incentive logic to the referral service and checkout flow per Pricing Gap Closure Spec B3-4.

**Business rules:**

1. **During early bird window (spots remain < 100):** No special incentive for the referred physician. They can still access early bird pricing if spots remain — that is the incentive.

2. **After early bird window closes (100 spots filled):** The referred physician receives their **first month free** at the standard rate ($279). This is implemented as a 100% Stripe coupon applied to the first invoice.

3. **NOT for clinic tier signups:** If the referred physician is signing up through a practice (clinic tier), they do NOT receive the first-month-free incentive. This applies only to individual subscription signups.

**Implementation:**

### Add to checkout session creation

Modify the existing `createCheckoutSession` function (in `platform.service.ts`) to check for referral eligibility:

```typescript
// Inside createCheckoutSession, after existing validation:

// Check if this user has a pending referral redemption
const pendingReferral = await findPendingByReferredUser(userId);

if (pendingReferral) {
  // Check if early bird window is closed
  const earlyBirdCount = await countEarlyBirdSubscriptions();
  const earlyBirdClosed = earlyBirdCount >= EARLY_BIRD_CAP; // 100

  // Only apply referee incentive if:
  // 1. Early bird window is closed
  // 2. Plan is NOT clinic tier
  // 3. Plan is NOT early bird (they're getting EB rate instead)
  const isClinicPlan = plan === 'CLINIC_MONTHLY' || plan === 'CLINIC_ANNUAL';
  const isEarlyBirdPlan = plan === 'EARLY_BIRD_MONTHLY' || plan === 'EARLY_BIRD_ANNUAL';

  if (earlyBirdClosed && !isClinicPlan && !isEarlyBirdPlan) {
    // Apply 100% coupon for first month
    // Add coupon to Stripe Checkout Session
    checkoutSessionParams.discounts = [{
      coupon: process.env.STRIPE_COUPON_REFERRAL_FIRST_MONTH_FREE,
    }];
  }
}
```

### Stripe Coupon Setup

The Stripe coupon must be pre-created in Stripe (not created dynamically). Add environment variable:
- `STRIPE_COUPON_REFERRAL_FIRST_MONTH_FREE` — ID of a Stripe coupon configured as:
  - 100% off
  - Duration: once (applies to first invoice only)
  - Currency: CAD
  - Name: "Referral - First Month Free"

Add this env var to the project's `.env.example` file with a comment.

### Helper function

```typescript
async function shouldApplyRefereeIncentive(
  referredUserId: string,
  plan: string
): Promise<boolean>
```

Returns `true` if:
- The user has a pending referral redemption
- The early bird window is closed (>= 100 early bird subscriptions)
- The plan is not a clinic tier plan
- The plan is not an early bird plan

This keeps the logic testable and separate from the checkout flow.

## Critical Security Rules

- The coupon is applied server-side during checkout session creation — the client cannot inject or modify coupon codes.
- The `STRIPE_COUPON_REFERRAL_FIRST_MONTH_FREE` env var must be set in production. If missing, log a warning and skip the coupon (do NOT crash).
- The early bird count check uses the same `countEarlyBirdSubscriptions()` that the early bird cap check uses — ensure consistency.
- Do NOT allow the referee incentive to stack with early bird pricing. If somehow both conditions are met, early bird takes precedence (but by design they are mutually exclusive since the incentive only activates after EB window closes).

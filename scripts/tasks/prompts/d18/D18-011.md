# Task D18-011: Repository: referral redemption CRUD operations

## What to Build

Add referral redemption repository functions alongside the referral code repository (in `apps/api/src/domains/platform/referral.repository.ts` or wherever D18-010 was placed).

**Functions to implement:**

### createRedemption(data: { referralCodeId: string, referrerUserId: string, referredUserId: string, anniversaryYear: number })
- Insert a new row into `referral_redemptions` with `status = 'PENDING'`
- `credit_month_value_cad`, `credit_applied_to`, `credit_applied_at`, `qualifying_event_at` are all NULL initially
- Returns the created `SelectReferralRedemption` record

### findPendingByReferredUser(referredUserId: string)
- Select from `referral_redemptions` where `referred_user_id = $1` and `status = 'PENDING'`
- Returns `SelectReferralRedemption | null`
- Used to check if a referred user already has a pending referral

### countQualifiedOrCreditedByReferrerAndYear(referrerUserId: string, anniversaryYear: number)
- Count rows in `referral_redemptions` where `referrer_user_id = $1` AND `anniversary_year = $2` AND `status IN ('QUALIFIED', 'CREDITED')`
- Returns `number`
- Used to enforce the 3-per-year cap

### updateRedemptionStatus(redemptionId: string, data: { status: string, creditMonthValueCad?: string, creditAppliedTo?: string, creditAppliedAt?: Date, qualifyingEventAt?: Date })
- Update `referral_redemptions` set the provided fields where `redemption_id = $1`
- Returns the updated record
- Used by service layer to transition between statuses (PENDING → QUALIFIED → CREDITED or EXPIRED)

### findRedemptionsByReferrer(referrerUserId: string, filters?: { status?: string, anniversaryYear?: number })
- Select from `referral_redemptions` where `referrer_user_id = $1`
- Apply optional filters for `status` and `anniversary_year`
- Order by `created_at DESC`
- Returns `SelectReferralRedemption[]`
- Used by GET /referrals/my-credits

### findPendingRedemptions()
- Select from `referral_redemptions` where `status = 'PENDING'`
- Returns `SelectReferralRedemption[]`
- Used by the `checkReferralQualification` scheduled job to find all pending redemptions

### findRedemptionById(redemptionId: string)
- Select from `referral_redemptions` where `redemption_id = $1`
- Returns `SelectReferralRedemption | null`
- Used by applyReferralCredit to load the redemption before applying

**Import the schema** from `packages/shared/src/schemas/db/platform.schema.ts` — use the redesigned `referralRedemptions` table (from D18-002).

## Critical Security Rules

- All queries must be parameterized via Drizzle ORM.
- `credit_month_value_cad` must be stored as `decimal(10,2)` — never use floating point for money.
- The `referrer_user_id` on redemptions must always match the `referrer_user_id` on the associated referral code. The service layer enforces this, but the repository should not allow mismatches if possible.

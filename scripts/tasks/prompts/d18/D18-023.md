# Task D18-023: Service: applyReferralCredit + clinic physician choice

## What to Build

Add the `applyReferralCredit` function and the clinic physician credit choice logic to the referral service.

**Function signature:**
```typescript
applyReferralCredit(redemptionId: string, userId: string, target?: CreditApplicationTarget): Promise<SelectReferralRedemption>
```

**Logic (Pricing Gap Closure Spec B3-2 and B3-3):**

### Credit Application

1. **Load the redemption:**
   - Call `findRedemptionById(redemptionId)`
   - If not found: throw `NotFoundError`
   - If `status !== 'QUALIFIED'`: throw `BusinessRuleError('Credit can only be applied to qualified redemptions', { code: 'NOT_QUALIFIED' })`
   - If `referrer_user_id !== userId`: throw `ForbiddenError('You can only apply your own referral credits')`

2. **Determine the target:**
   - If referrer is on a **clinic tier** (CLINIC_MONTHLY or CLINIC_ANNUAL):
     - `target` parameter is required — must be `PRACTICE_INVOICE` or `INDIVIDUAL_BANK`
     - If not provided and this is an explicit user action: throw `BusinessRuleError('Clinic physicians must choose a credit target', { code: 'TARGET_REQUIRED' })`
   - If referrer is on an **individual plan**:
     - `target` is ignored — credit is always applied to the individual's next invoice
     - Set `credit_applied_to = null` (not applicable for individual plans, or you can set a sensible default)

3. **Apply the credit based on target:**

   **Option A — PRACTICE_INVOICE:**
   - Look up the referrer's active `practice_memberships` to find their `practice_id`
   - Look up the practice's `stripe_subscription_id`
   - Create a Stripe credit note or negative invoice item on the practice's next invoice:
     ```typescript
     stripe.invoiceItems.create({
       customer: practiceStripeCustomerId,
       amount: -Math.round(creditMonthValueCad * 100), // negative cents
       currency: 'cad',
       description: `Referral credit - 1 month free (${referralCode})`,
     });
     ```
   - Set `credit_applied_to = 'PRACTICE_INVOICE'`

   **Option B — INDIVIDUAL_BANK:**
   - Do NOT apply to any invoice now
   - Set `credit_applied_to = 'INDIVIDUAL_BANK'`
   - The credit is held and will be applied when/if the physician moves to an individual subscription
   - Add a `// TODO: Apply banked credits when physician transitions to individual billing` comment for the future integration

   **Individual plan (no choice needed):**
   - Look up the referrer's `stripe_customer_id` from their subscription
   - Create a Stripe negative invoice item on the referrer's next invoice (same as Option A but on the individual's Stripe customer)

4. **Update the redemption:**
   - Call `updateRedemptionStatus(redemptionId, { status: 'CREDITED', creditAppliedTo: target, creditAppliedAt: new Date() })`

5. **Return the updated record.**

### Default After 7 Days (Scheduled Job)

Add a function:
```typescript
applyDefaultCreditChoice(): Promise<number>
```

- Find all redemptions where `status = 'QUALIFIED'` AND `qualifying_event_at <= now() - 7 days` AND referrer is on a clinic tier
- For each: call `applyReferralCredit(redemptionId, referrerUserId, 'PRACTICE_INVOICE')`
- This auto-defaults to Option A (practice invoice) after 7 days of no response
- Import `REFERRAL_CREDIT_CHOICE_DEADLINE_DAYS` (7) from constants

### Immutability

- Once `status = 'CREDITED'` and `credit_applied_to` is set, neither can be changed
- The `applyReferralCredit` function must reject any call where the redemption is already CREDITED
- Add a check: `if (redemption.status === 'CREDITED') throw BusinessRuleError('Credit has already been applied', { code: 'ALREADY_CREDITED' })`

### Annual Subscriber Note

For annual subscribers, the credit is a monetary reduction on their next annual renewal invoice (not a separate payment). The Stripe negative invoice item achieves this — when the next invoice is generated, the credit is automatically deducted.

## Critical Security Rules

- **Authorization:** Only the referrer (owner of the credit) can choose the application target. Validate `referrer_user_id === userId` from the authenticated session.
- **Immutability:** Once a credit is applied (CREDITED status), it cannot be changed. This is a hard business rule.
- **Financial precision:** All monetary calculations use `decimal` — never floating point. Stripe amounts are in cents (multiply by 100, round to integer).
- **No PHI sent to Stripe:** The invoice item description contains only the referral code, not physician names or patient data.
- **Audit logging:** Log credit application events (who, when, which target, amount) for financial audit trail.

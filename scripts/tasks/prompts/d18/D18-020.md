# Task D18-020: Service: generateReferralCode

## What to Build

Add the `generateReferralCode` function to the platform service layer. Create a new file `apps/api/src/domains/platform/referral.service.ts` (or add to existing `platform.service.ts` — follow the pattern established in the project for the platform domain).

**Function signature:**
```typescript
generateReferralCode(userId: string): Promise<SelectReferralCode>
```

**Logic:**

1. Check if the user already has an active referral code (call `findReferralCodeByUserId`). If yes, return the existing code.

2. Generate a unique 8-character alphanumeric code:
   - Characters: `ABCDEFGHJKLMNPQRSTUVWXYZ23456789` (exclude 0/O/1/I/L to avoid ambiguity)
   - Use `crypto.randomBytes()` or equivalent for randomness — NOT `Math.random()`
   - Length: 8 characters (per `REFERRAL_CODE_LENGTH` constant from D18-003)

3. Check for uniqueness: call `findReferralCodeByCode` with the generated code. If collision, regenerate (retry up to 10 times, then throw an error).

4. Call `createReferralCode(userId, code)` to persist.

5. Return the created record.

**Auto-generation at signup:**

This function should also be called automatically when a physician's subscription becomes active. Add a hook or call to `generateReferralCode` in the subscription activation flow (likely in the Stripe webhook handler for `checkout.session.completed` or `invoice.paid`). Look at the existing `platform.service.ts` to find where subscription activation happens, and add the call there.

If the subscription activation path is not clear, add a `// TODO: Call generateReferralCode(userId) on subscription activation` comment at the top of the file and document the integration point in the function's JSDoc.

**Import constants:**
- `REFERRAL_CODE_LENGTH` from `packages/shared/src/constants/platform.constants.ts`

```typescript
  it('generates a new 8-character alphanumeric code');
  it('generated code contains only valid characters (no 0/O/1/I/L)');
  it('persists the code via createReferralCode');
  it('retries on collision and generates a different code');
  it('throws error after 10 failed uniqueness attempts');
  it('uses cryptographically secure random bytes, not Math.random');
});
```

## Critical Security Rules

- Use `crypto.randomBytes()` for code generation — codes must be unpredictable.
- Do NOT use `Math.random()` — it is not cryptographically secure and codes could be predicted.
- The `userId` must come from the authenticated session, never from request input.
- Rate limit code generation (one code per user — enforced by checking for existing code first).

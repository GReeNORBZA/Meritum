# Task D18-001: Redesign referral_codes table

## What to Build

**REPLACE** the existing `referral_codes` table in `packages/shared/src/schemas/db/platform.schema.ts`. The old schema uses a $50/10-per-year referral model that is being replaced by a 1-month-free/3-per-year model per the Pricing Gap Closure Spec (B3-1).

**Remove these columns entirely:**
- `redemption_count` (INTEGER) — now calculated, not stored
- `max_redemptions` (INTEGER) — cap is now 3/year and enforced in service layer

**Rename this column:**
- `physician_user_id` → `referrer_user_id` (same type: UUID FK → users)

**Change this default:**
- `is_active` default from `false` → `true` (codes are active from creation, not dormant)

**Remove the old MVP accommodation comments** (`// MVP accommodation: schema defined, not actively used` and the `$50 credit per referral, max 10 per physician per year` comment).

The resulting `referral_codes` table must have exactly these columns:

| Column | Drizzle Definition | Constraints |
|--------|-------------------|-------------|
| referral_code_id | `uuid('referral_code_id').primaryKey().defaultRandom()` | PK |
| referrer_user_id | `uuid('referrer_user_id').notNull().references(() => users.userId)` | FK → users |
| code | `varchar('code', { length: 20 }).notNull()` | Unique (via index) |
| is_active | `boolean('is_active').notNull().default(true)` | Default **true** |
| created_at | `timestamp('created_at', { withTimezone: true }).notNull().defaultNow()` | |

**Indexes (keep/update):**
- `uniqueIndex('referral_codes_code_idx').on(table.code)` — keep as-is
- `index('referral_codes_referrer_user_id_idx').on(table.referrerUserId)` — rename from `physician_user_id` index

**Update the inferred types** at the bottom of the file:
- `InsertReferralCode` and `SelectReferralCode` must reflect the new schema

**Important:** This is a REPLACEMENT of the existing table definition, not an extension. Delete the old column definitions and write the new ones.

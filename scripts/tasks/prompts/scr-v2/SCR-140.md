# Task SCR-140: Fix fetchWithRetry to cover response body transfer in timeout

## What to Build

Fix `fetchWithRetry` in `scripts/lib/fee-navigator-utils.ts` (line ~66):

**Problem:** `clearTimeout(timer)` is called at line 74 after `await fetch()` resolves (headers received). But `await resp.text()` at line 87 can still hang if the body stalls.

**Fix:** Move `clearTimeout(timer)` to AFTER `await resp.text()` completes. Keep clearTimeout in error/retry branches.

```
const resp = await fetch(url, { ...options, signal: controller.signal, ... });
// Don't clearTimeout here — body may still be streaming
if (resp.status === 429 || resp.status === 503) { clearTimeout(timer); /* backoff */ continue; }
if (!resp.ok) { clearTimeout(timer); throw ...; }
const body = await resp.text(); // Timer still running — abort on stall
clearTimeout(timer); // NOW safe to clear
```

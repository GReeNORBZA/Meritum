# Task D16-025: Service: getPracticeSeats and getPracticeInvoice — ZERO PHI, return only name/email/joinedAt/billingMode

## What to Build

Add `getPracticeSeats()` and `getPracticeInvoice()` functions to `apps/api/src/domains/platform/practice.service.ts`. Per Pricing Gap Closure Spec B1-7 and B1-11.

### getPracticeSeats

**Function signature:**
```typescript
async function getPracticeSeats(
  practiceId: string,
  adminUserId: string,
): Promise<PracticeSeat[]>
```

**Return type (define this interface):**
```typescript
interface PracticeSeat {
  physicianName: string;    // Full name from users table
  email: string;            // Email from users table
  joinedAt: Date;           // From practice_memberships
  billingMode: string;      // 'PRACTICE_CONSOLIDATED' or 'INDIVIDUAL_EARLY_BIRD'
}
```

**Business logic:**

1. **Validate caller is PRACTICE_ADMIN for this practice:**
   - Find practice, verify `practice.admin_user_id === adminUserId`
   - If not, throw `ForbiddenError`

2. **Get active memberships:**
   - Call `membershipRepo.findActiveMembershipsByPracticeId(practiceId)`

3. **For each membership, join with users table to get name and email:**
   - Use a repository method or inline query to get user details
   - **ONLY select:** `user.full_name` (or first_name + last_name), `user.email`

4. **Map to PracticeSeat objects:**
   - Return ONLY: `physicianName`, `email`, `joinedAt`, `billingMode`

5. **DATA ISOLATION — the returned data MUST NOT include any of the following:**
   - Claim counts or claim data
   - Billing volumes or revenue
   - Rejection rates
   - Patient counts or patient data
   - Analytics data of any kind
   - AI suggestion data
   - Individual payment history
   - Subscription plan or pricing details of individual physicians
   - Any data from: `claims`, `ahcip_claim_details`, `wcb_claim_details`, `patients`, `analytics_cache`, `generated_reports`, `ai_suggestion_events`, `payment_history`

6. **This is enforced at the repository layer**, not just the handler layer. The query itself must only join `practice_memberships` with `users`. No other tables.

### getPracticeInvoice

**Function signature:**
```typescript
async function getPracticeInvoice(
  practiceId: string,
  adminUserId: string,
): Promise<PracticeInvoiceInfo>
```

**Return type (define this interface):**
```typescript
interface PracticeInvoiceInfo {
  totalAmount: string;          // Total practice invoice amount (CAD)
  perSeatRate: string;          // Per-seat rate (e.g., '251.10')
  consolidatedSeatCount: number; // Number of PRACTICE_CONSOLIDATED seats
  totalHeadcount: number;        // Total active members (all billing modes)
  billingFrequency: string;      // 'MONTHLY' or 'ANNUAL'
  nextInvoiceDate: Date;         // Next billing date
  gstAmount: string;             // GST on consolidated amount
}
```

**Business logic:**

1. **Validate caller is PRACTICE_ADMIN for this practice:**
   - Same validation as above

2. **Get practice details and Stripe subscription info:**
   - Use Stripe API to get upcoming invoice for the practice's Stripe subscription
   - OR calculate from: consolidatedSeatCount * perSeatRate

3. **Calculate amounts:**
   - `consolidatedSeatCount`: count of `PRACTICE_CONSOLIDATED` members
   - `perSeatRate`: from `SubscriptionPlanPricing[CLINIC_MONTHLY].amount` or `CLINIC_ANNUAL` (divided by 12 for display)
   - `totalAmount`: consolidatedSeatCount * perSeatRate
   - `gstAmount`: totalAmount * GST_RATE

4. **This returns ONLY the practice's consolidated invoice data.** It does NOT return individual physician payment records.

## Critical Security Rules

**THIS IS THE MOST CRITICAL SECURITY REQUIREMENT IN THE ENTIRE CLINIC TIER.**

Per Pricing Gap Closure Spec B1-11:

- `getPracticeSeats()` returns ONLY: `{ physicianName, email, joinedAt, billingMode }` per seat. **Nothing else. No exceptions.**
- The `billingMode` field is practice account management data, not PHI. It tells the admin which physicians are on consolidated billing vs individual early bird.
- No practice admin endpoint returns data from: `claims`, `ahcip_claim_details`, `wcb_claim_details`, `patients`, `analytics_cache`, `generated_reports`, `ai_suggestion_events`, `payment_history` (individual physician payments).
- The practice admin sees the PRACTICE invoice (consolidated), not individual physician payment records.
- **This is enforced at the repository layer, not the handler layer.** The SQL query must only join `practice_memberships` with `users`. It must NEVER join with any PHI-containing table.
- Per the pricing strategy document Section 2.4: "Individual physician billing data is personal professional information. Exposing it to a practice administrator would create a surveillance dynamic."

# Task D16-024: Service: handleEndOfMonthRemovals scheduled job — process removals, check headcount, dissolve practice if below 5

## What to Build

Add the `handleEndOfMonthRemovals()` function to `apps/api/src/domains/platform/practice.service.ts`. Per Pricing Gap Closure Spec B1-7.

**Function signature:**
```typescript
async function handleEndOfMonthRemovals(): Promise<{
  processedCount: number;
  dissolvedPractices: string[];
}>
```

This is a **scheduled job** that runs at the end of each calendar month (or daily, checking for pending removals).

**Business logic:**

1. **Find all pending removals:**
   - Call `membershipRepo.findPendingRemovals(new Date())`
   - These are memberships where `removal_effective_at <= now` AND `is_active = true`

2. **Group removals by practice:**
   - Group the pending removals by `practice_id` for batch processing

3. **For each practice with pending removals:**

   a. **Process each removal:**
      - Call `membershipRepo.deactivateMembership(membership.membership_id)` — sets `is_active = false`
      - If the removed physician was `PRACTICE_CONSOLIDATED`:
        - Decrement practice Stripe subscription quantity by 1
        - Call `stripe.subscriptions.update(stripeSubId, { quantity: newQuantity })`
      - If the removed physician was `INDIVIDUAL_EARLY_BIRD`:
        - No Stripe quantity change (they were never on the practice subscription)
        - Their individual subscription continues unchanged

   b. **Check remaining headcount (after all removals for this practice):**
      - Call `practiceRepo.getActiveHeadcount(practiceId)`
      - This counts ALL active members (both billing modes)

   c. **If headcount < CLINIC_MINIMUM_PHYSICIANS (5):**
      **Dissolve the practice:**

      i. **Cancel practice Stripe subscription:**
         - `stripe.subscriptions.cancel(practice.stripe_subscription_id)`

      ii. **Transition PRACTICE_CONSOLIDATED members to individual subscriptions:**
         - For each remaining `PRACTICE_CONSOLIDATED` member:
           - Create a new individual subscription at STANDARD rate
           - Plan: `STANDARD_MONTHLY` if practice was MONTHLY, `STANDARD_ANNUAL` if ANNUAL
           - Create Stripe subscription for the individual physician
           - Update the `subscriptions` table with new individual subscription

      iii. **INDIVIDUAL_EARLY_BIRD members keep their existing subscriptions:**
          - No changes needed — they already have individual Stripe subscriptions

      iv. **Deactivate all remaining memberships:**
         - Call `membershipRepo.deactivateAllMemberships(practiceId)`

      v. **Update practice status:**
         - Call `practiceRepo.updatePracticeStatus(practiceId, 'CANCELLED')`

      vi. **Emit notifications to all affected physicians:**
         - For PRACTICE_CONSOLIDATED members: "Your practice has been dissolved due to insufficient members. You have been transitioned to an individual Standard subscription at $279/month (or $3,181/year)."
         - For INDIVIDUAL_EARLY_BIRD members: "Your practice has been dissolved. Your individual early bird subscription remains unchanged."
         - For the admin: "Your practice has been dissolved. Clinic tier pricing has been removed."

      vii. **Audit log:**
          - Log action: `'practice.dissolved'`
          - Include: practice ID, remaining member count, reason: 'BELOW_MINIMUM_HEADCOUNT'

4. **Return summary:**
   - `processedCount`: total memberships deactivated
   - `dissolvedPractices`: array of practice IDs that were dissolved

## Critical Security Rules

- Notifications must NOT contain PHI — only practice name, physician name, plan details, and effective dates.
- Stripe operations must be idempotent where possible. If the job runs twice, it should not double-process.
- The dissolution flow must be transactional — if creating individual subscriptions fails partway, roll back.
- The headcount check must count ALL active members (both billing modes) to determine if the 5-physician minimum is met.
- Stripe subscription quantity only reflects `PRACTICE_CONSOLIDATED` members, but headcount for the minimum check includes `INDIVIDUAL_EARLY_BIRD` members too.

# Task D16-023: Service: removePhysician — validate PRACTICE_ADMIN, set removal_effective_at, emit notification if below 5

## What to Build

Add the `removePhysician()` function to `apps/api/src/domains/platform/practice.service.ts`. Per Pricing Gap Closure Spec B1-7.

**Function signature:**
```typescript
async function removePhysician(
  practiceId: string,
  physicianUserId: string,
  removedByUserId: string,
): Promise<void>
```

**Business logic:**

1. **Validate the remover is PRACTICE_ADMIN for this practice:**
   - Find the practice by `practiceId`
   - Verify `practice.admin_user_id === removedByUserId`
   - If not, throw `ForbiddenError` with code `NOT_PRACTICE_ADMIN`

2. **Cannot remove the admin themselves:**
   - If `physicianUserId === practice.admin_user_id`, throw `BusinessRuleError` with code `CANNOT_REMOVE_ADMIN`
   - The admin must transfer admin role first or dissolve the practice entirely

3. **Find the active membership:**
   - Call `membershipRepo.findMembershipByPracticeAndPhysician(practiceId, physicianUserId)`
   - If not found, throw `NotFoundError` with code `MEMBERSHIP_NOT_FOUND`

4. **Check if removal is already scheduled:**
   - If `membership.removal_effective_at` is already set, throw `BusinessRuleError` with code `REMOVAL_ALREADY_SCHEDULED`

5. **Calculate removal effective date:**
   - `removal_effective_at` = end of current calendar month (not billing cycle)
   - Example: If today is Feb 16, `removal_effective_at` = Feb 28 23:59:59 UTC
   - Use: last day of current month, 23:59:59.999 UTC

6. **Schedule the removal:**
   - Call `membershipRepo.setRemovalScheduled(membership.membership_id, new Date(), removalEffectiveAt)`
   - Do NOT set `is_active = false` — the physician retains access until end of month

7. **Check headcount after this removal takes effect:**
   - Get current active headcount: `practiceRepo.getActiveHeadcount(practiceId)`
   - Subtract 1 (for the member being removed)
   - If projected headcount < `CLINIC_MINIMUM_PHYSICIANS` (5):
     - Emit notification to admin: "Warning: removing this physician will drop your practice below the 5-physician minimum. Clinic tier pricing will be lost at end of month."
     - Log warning in audit

8. **Emit notification to the removed physician:**
   - Notify them of their removal and the effective date
   - Follow existing notification pattern

9. **Audit log:**
   - Log action: `'practice.physician_removed'`
   - Include: practice ID, physician user ID, removed by user ID, removal effective date

## Critical Security Rules

- Only PRACTICE_ADMIN can remove physicians. This is enforced at the service layer AND the route layer.
- The admin cannot remove themselves — this prevents accidental practice orphaning.
- Removal is delayed until end of month to ensure fair billing. The physician retains full access until then.
- Notifications must NOT contain PHI — only the practice name, physician name, and effective date.

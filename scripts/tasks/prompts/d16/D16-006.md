# Task D16-006: Zod validation schemas for practice management endpoints + generate migration

## What to Build

This task has two parts. Per Pricing Gap Closure Spec B1-1 through B1-8.

### Part 1: Add Zod validation schemas

**File:** `packages/shared/src/schemas/platform.schema.ts`

Add the following Zod schemas after the existing schemas:

**Create Practice:**
```typescript
export const createPracticeSchema = z.object({
  name: z.string().min(1).max(200).trim(),
  billing_frequency: z.enum(['MONTHLY', 'ANNUAL']),
});

export type CreatePractice = z.infer<typeof createPracticeSchema>;
```

**Update Practice:**
```typescript
export const updatePracticeSchema = z.object({
  name: z.string().min(1).max(200).trim().optional(),
  billing_frequency: z.enum(['MONTHLY', 'ANNUAL']).optional(),
});

export type UpdatePractice = z.infer<typeof updatePracticeSchema>;
```

**Invite Physician:**
```typescript
export const invitePhysicianSchema = z.object({
  email: z.string().email().max(255).toLowerCase().trim(),
});

export type InvitePhysician = z.infer<typeof invitePhysicianSchema>;
```

**Accept Invitation (token param):**
```typescript
export const acceptInvitationParamsSchema = z.object({
  token: z.string().min(1).max(256),
});

export type AcceptInvitationParams = z.infer<typeof acceptInvitationParamsSchema>;
```

**Practice ID param:**
```typescript
export const practiceIdParamsSchema = z.object({
  id: z.string().uuid(),
});

export type PracticeIdParams = z.infer<typeof practiceIdParamsSchema>;
```

**Practice seat removal params:**
```typescript
export const removeSeatParamsSchema = z.object({
  id: z.string().uuid(),
  userId: z.string().uuid(),
});

export type RemoveSeatParams = z.infer<typeof removeSeatParamsSchema>;
```

**Practice seats query (minimal — no filters for MVP):**
```typescript
export const practiceSeatsQuerySchema = z.object({
  page: z.coerce.number().int().min(1).default(1),
  page_size: z.coerce.number().int().min(1).max(100).default(50),
});

export type PracticeSeatsQuery = z.infer<typeof practiceSeatsQuerySchema>;
```

**Practice invoices query:**
```typescript
export const practiceInvoicesQuerySchema = z.object({
  page: z.coerce.number().int().min(1).default(1),
  page_size: z.coerce.number().int().min(1).max(50).default(20),
});

export type PracticeInvoicesQuery = z.infer<typeof practiceInvoicesQuerySchema>;
```

Also update the `createCheckoutSessionSchema` plan enum to include the new clinic plans:
```typescript
export const createCheckoutSessionSchema = z.object({
  plan: z.enum([
    SubscriptionPlan.STANDARD_MONTHLY,
    SubscriptionPlan.STANDARD_ANNUAL,
    SubscriptionPlan.EARLY_BIRD_MONTHLY,

  success_url: z.string().url(),
```

Import `SubscriptionPlan` — it is already imported but may need the updated version after D16-001 adds the new plan constants.

### Part 2: Generate database migration

After confirming the shared package builds, generate the Drizzle migration:

```bash
cd apps/api && pnpm drizzle-kit generate 2>&1
```

This will generate a migration file for the new tables (practices, practice_memberships, practice_invitations) and the modified subscriptions table (new practice_id column).

## Critical Security Rules

- Input validation is the first line of defense. All string inputs are trimmed and length-limited.
- Email addresses are normalized to lowercase.
- UUID parameters are validated as proper UUIDs to prevent injection.
- Practice name has a 200-character max to prevent abuse.

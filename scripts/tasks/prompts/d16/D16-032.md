# Task D16-032: Integration tests for all practice routes

## What to Build

Create `apps/api/test/integration/platform/practice.integration.test.ts` with comprehensive integration tests for all 8 practice routes. Per Pricing Gap Closure Spec B1-8.

**Test setup:**
- Use the existing integration test infrastructure (test database, test server, session helpers)
- Follow the patterns established in existing integration tests (e.g., `test/integration/platform/`)
- Create test users with appropriate roles (PHYSICIAN, PRACTICE_ADMIN)
- Use test helpers to create authenticated sessions

**Test suites:**

### POST /api/v1/practices
```typescript
describe('POST /api/v1/practices', () => {
  it('creates a practice and returns 201 with practice data');
  it('assigns PRACTICE_ADMIN role to the creating physician');
  it('creates a membership for the admin');
  it('returns 401 without authentication');
  it('returns 403 for non-PHYSICIAN role');
  it('returns 422 if user already admins a practice');
  it('returns 422 if user is on another practice');
  it('returns 400 for missing name');
  it('returns 400 for invalid billing_frequency');
  it('returns 400 for name exceeding 200 characters');
});
```

### GET /api/v1/practices/:id
```typescript
describe('GET /api/v1/practices/:id', () => {
  it('returns practice details for PRACTICE_ADMIN');
  it('returns 401 without authentication');
  it('returns 403 for non-admin physician');
  it('returns 403 for admin of different practice');
  it('returns 404 for non-existent practice');
});
```

### GET /api/v1/practices/:id/seats
```typescript
describe('GET /api/v1/practices/:id/seats', () => {
  it('returns list of seats with physicianName, email, joinedAt, billingMode');
  it('response contains ONLY physicianName, email, joinedAt, billingMode per seat');
  it('response does NOT contain any claim data fields');
  it('response does NOT contain any billing volume fields');
  it('response does NOT contain any patient data fields');
  it('includes both PRACTICE_CONSOLIDATED and INDIVIDUAL_EARLY_BIRD members');

  it('returns 401 without authentication');
  it('returns 403 for non-admin');
});
```

### POST /api/v1/practices/:id/invitations

```typescript
describe('POST /api/v1/practices/:id/invitations', () => {
```

### POST /api/v1/practice-invitations/:token/accept

## Critical Security Rules

- **DATA ISOLATION:** Integration tests MUST verify that the seats endpoint returns ONLY the 4 permitted fields. Use `Object.keys()` or similar to assert no extra keys are present.
- Integration tests MUST verify cross-practice isolation: admin of practice A gets 403 on practice B endpoints.
- Integration tests MUST verify the invitation response does NOT contain the raw token.
- Integration tests MUST verify the invoices endpoint returns practice-level data, NOT individual physician payment records.

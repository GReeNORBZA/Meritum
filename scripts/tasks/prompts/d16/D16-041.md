# Task D16-041: Security: authz — only PRACTICE_ADMIN can manage seats/invoices, cross-practice isolation

## What to Build

Create `apps/api/test/security/platform/practice.authz.security.ts` with authorization enforcement tests. Per Pricing Gap Closure Spec B1-10.

**Test setup — create these test actors:**

1. **practiceAdminA** — PRACTICE_ADMIN for Practice A (also has PHYSICIAN role)
2. **practiceAdminB** — PRACTICE_ADMIN for Practice B (also has PHYSICIAN role)
3. **physicianOnPracticeA** — Regular PHYSICIAN who is a member of Practice A (NOT admin)
4. **physicianNoPractice** — Regular PHYSICIAN not on any practice
5. **delegateUser** — DELEGATE role only

**Test suites:**

### Only PRACTICE_ADMIN can access management endpoints

```typescript
describe('PRACTICE_ADMIN authorization', () => {
  // A regular PHYSICIAN on the practice (not admin) gets 403
  it('GET /practices/:id returns 403 for non-admin physician on the practice');
  it('PATCH /practices/:id returns 403 for non-admin physician');
  it('GET /practices/:id/seats returns 403 for non-admin physician');
  it('POST /practices/:id/invitations returns 403 for non-admin physician');
  it('DELETE /practices/:id/seats/:userId returns 403 for non-admin physician');
  it('GET /practices/:id/invoices returns 403 for non-admin physician');

  // A PHYSICIAN not on any practice gets 403
  it('GET /practices/:id returns 403 for physician not on practice');
  it('GET /practices/:id/seats returns 403 for physician not on practice');
  it('GET /practices/:id/invoices returns 403 for physician not on practice');

  // A DELEGATE gets 403
  it('GET /practices/:id returns 403 for DELEGATE role');
  it('GET /practices/:id/seats returns 403 for DELEGATE role');
  it('POST /practices/:id/invitations returns 403 for DELEGATE role');
});
```

### Cross-practice isolation

```typescript
describe('Cross-practice isolation', () => {
  // Admin of Practice A cannot access Practice B
  it('GET /practices/:practiceB returns 403 for adminA');
  it('PATCH /practices/:practiceB returns 403 for adminA');
  it('GET /practices/:practiceB/seats returns 403 for adminA');
  it('POST /practices/:practiceB/invitations returns 403 for adminA');
  it('DELETE /practices/:practiceB/seats/:userId returns 403 for adminA');
  it('GET /practices/:practiceB/invoices returns 403 for adminA');

  // Admin of Practice B cannot access Practice A
  it('GET /practices/:practiceA returns 403 for adminB');
  it('GET /practices/:practiceA/seats returns 403 for adminB');
  it('GET /practices/:practiceA/invoices returns 403 for adminB');
});
```

## Critical Security Rules

- PRACTICE_ADMIN role is scoped to a SPECIFIC practice. Admin of practice A has ZERO access to practice B.
- Regular physicians on a practice (non-admin) cannot access management endpoints (seats, invoices, invitations, settings).
- DELEGATE role has NO access to any practice endpoint.
- 403 responses must not leak whether a practice exists or its details. The error message should be generic.
- The authorization check must happen BEFORE any data is loaded/returned.

# Task D16-043: Security: input — SQL injection/XSS on practice name, invitation email

## What to Build

Create `apps/api/test/security/platform/practice.input.security.ts` with input validation and injection prevention tests. Per Pricing Gap Closure Spec B1-10.

**Test setup:**
- Create a practice admin with an active practice
- Test each user-controlled input field for injection attacks

### SQL Injection Tests

```typescript
describe('SQL Injection Prevention', () => {
  describe('Practice name', () => {
    it('POST /practices rejects name with SQL injection: Robert\'); DROP TABLE practices;--');
    it('POST /practices rejects name with UNION SELECT: a\' UNION SELECT * FROM users--');
    it('PATCH /practices/:id rejects name with SQL injection');
    it('practice name with single quotes is safely escaped and stored');
    it('practice name with double quotes is safely escaped and stored');
    it('practice name with semicolons is safely escaped and stored');
    it('practice name with backslashes is safely escaped and stored');
  });

  describe('Invitation email', () => {
    it('POST /invitations rejects email with SQL injection: test@test.com\'; DROP TABLE users;--');
    it('POST /invitations rejects email with UNION SELECT payload');
    it('POST /invitations rejects non-email strings');
  });

  describe('Path parameters', () => {
    it('GET /practices/:id rejects non-UUID practice ID');
    it('GET /practices/:id rejects SQL in practice ID: 1 OR 1=1');
    it('DELETE /practices/:id/seats/:userId rejects non-UUID userId');
    it('POST /practice-invitations/:token/accept rejects SQL in token');
  });
});
```

### XSS Prevention Tests

```typescript
describe('XSS Prevention', () => {
  describe('Practice name', () => {
    it('POST /practices stores name without executing script tags');
    it('practice name <script>alert("xss")</script> is escaped in response');
    it('practice name with event handlers: <img onerror="alert(1)"> is escaped');
    it('practice name with encoded entities is handled safely');
  });

  describe('Invitation email', () => {
    it('POST /invitations rejects email with HTML: <script>alert(1)</script>@test.com');
  });

  describe('Responses', () => {
    it('GET /practices/:id returns Content-Type application/json (not text/html)');
    it('GET /practices/:id/seats returns Content-Type application/json');
    it('practice name stored with special characters is returned as-is in JSON (no HTML rendering)');
  });
});
```

## Critical Security Rules

- All input validation is done via Zod schemas BEFORE reaching business logic.
- Drizzle ORM parameterized queries prevent SQL injection at the database layer.
- JSON responses with `Content-Type: application/json` prevent XSS in API responses.
- Practice names with special characters must be stored and returned safely — no sanitization that alters the name, but proper escaping in all contexts.
- UUID validation on path parameters prevents injection via URL manipulation.

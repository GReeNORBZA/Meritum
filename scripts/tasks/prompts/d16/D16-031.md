# Task D16-031: Practice handlers ظ¤ thin layer connecting routes to service

## What to Build

Create `apps/api/src/domains/platform/practice.handlers.ts` with handler functions for all practice routes. Per Pricing Gap Closure Spec B1-8.

Each handler is a thin layer that:
1. Extracts validated input from the request (params, body, query)
2. Gets the authenticated user ID from the session/context
3. Calls the appropriate service function
4. Returns the formatted response

**Handler functions:**

**createPracticeHandler:**
```typescript
- Extract: body.name, body.billing_frequency
- Get: session.userId
- Call: practiceService.createPractice(userId, name, billingFrequency)
- Return: 201, { practice: result }
```

**getPracticeHandler:**
```typescript
- Extract: params.id
- Get: session.userId
- Call: practiceService.getPractice(params.id, userId)
- Return: 200, { practice: result }
```

**updatePracticeHandler:**
```typescript
- Extract: params.id, body (name?, billing_frequency?)
- Get: session.userId
- Call: practiceService.updatePractice(params.id, userId, body)
- Return: 200, { practice: result }
```

**getPracticeSeatsHandler:**
```typescript
- Extract: params.id, query (page, page_size)
- Get: session.userId
- Call: practiceService.getPracticeSeats(params.id, userId)
- Return: 200, { seats: result }
- CRITICAL: Do NOT add any extra fields to the response. Return ONLY what the service returns.
```

**invitePhysicianHandler:**
```typescript
- Extract: params.id, body.email
- Get: session.userId
- Call: practiceService.invitePhysician(params.id, body.email, userId)
- Return: 201, { invitation: { invitationId, invitedEmail, status, expiresAt } }
- CRITICAL: Do NOT return the raw token in the response. The token is sent via email only.
```

**acceptInvitationHandler:**
```typescript
- Extract: params.token
- Get: session.userId
- Call: practiceService.acceptInvitation(params.token, userId)
- Return: 200, { membership: { membershipId, practiceId, billingMode, joinedAt } }
```

**removePhysicianHandler:**
```typescript
- Extract: params.id (practiceId), params.userId (physicianUserId)
- Get: session.userId
- Call: practiceService.removePhysician(params.id, params.userId, userId)
- Return: 200, { message: 'Physician removal scheduled for end of month', removalEffectiveAt }
```

**getPracticeInvoicesHandler:**
```typescript
- Extract: params.id, query (page, page_size)
- Get: session.userId
- Call: practiceService.getPracticeInvoice(params.id, userId)
- Return: 200, { invoice: result }
```

**Error handling:**

## Critical Security Rules

- Handlers are thin ظ¤ they MUST NOT contain business logic. All logic lives in the service layer.
- **DATA ISOLATION:** The seats handler MUST NOT enrich the response with additional data. It returns EXACTLY what `getPracticeSeats()` returns: `physicianName`, `email`, `joinedAt`, `billingMode`. No claim data, no billing volumes, no PHI.
- The invitation response MUST NOT include the raw token. Only: `invitationId`, `invitedEmail`, `status`, `expiresAt`.
- Error responses MUST NOT leak internal details (Stripe IDs, user IDs of other members, etc.) to non-admins.

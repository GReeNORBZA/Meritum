# Task D16-010: Practice repository — CRUD for practices, membership queries, headcount checks

## What to Build

Create `apps/api/src/domains/platform/practice.repository.ts` with all practice CRUD operations. Per Pricing Gap Closure Spec B1-1 and B1-7.

**Functions to implement:**

**createPractice(data: InsertPractice): Promise\<SelectPractice\>**
- Insert into `practices` table
- Return the created record

**findPracticeById(practiceId: string): Promise\<SelectPractice | null\>**
- Find practice by primary key
- Return null if not found

**findPracticeByAdminUserId(adminUserId: string): Promise\<SelectPractice | null\>**
- Find practice where `admin_user_id` matches
- Return only ACTIVE practices (status = 'ACTIVE')

**updatePractice(practiceId: string, data: Partial\<InsertPractice\>): Promise\<SelectPractice\>**
- Update the practice record
- Set `updated_at` to now
- Return the updated record

**updatePracticeStatus(practiceId: string, status: string): Promise\<void\>**
- Update the practice status field
- Set `updated_at` to now

**updatePracticeStripeIds(practiceId: string, stripeCustomerId: string, stripeSubscriptionId: string): Promise\<void\>**
- Update both Stripe ID fields
- Set `updated_at` to now

**getActiveHeadcount(practiceId: string): Promise\<number\>**
- Count ALL active memberships (is_active = true) for the practice
- This counts BOTH billing modes — `PRACTICE_CONSOLIDATED` and `INDIVIDUAL_EARLY_BIRD`
- Used to check the 5-physician minimum

**getConsolidatedSeatCount(practiceId: string): Promise\<number\>**
- Count active memberships where billing_mode = 'PRACTICE_CONSOLIDATED'
- This is the Stripe subscription quantity

**findActivePractices(): Promise\<SelectPractice[]\>**
- Return all practices with status = 'ACTIVE'

**Use the Drizzle ORM `db` instance** — import it from the shared database module following the pattern used in existing repositories (e.g., `apps/api/src/domains/platform/platform.repository.ts`).

**Import the schema tables** from `@meritum/shared/schemas/db/platform.schema` (or the appropriate import path matching the existing codebase pattern).

## Critical Security Rules

- Repository layer returns raw data. It does NOT filter PHI — that is done at the service layer.
- No claim data, patient data, or analytics data is ever queried from the practice repository.
- All queries use parameterized inputs via Drizzle ORM — no raw SQL string concatenation.

# Task D16-040: Security: authn — every practice route returns 401 without session

## What to Build

Create `apps/api/test/security/platform/practice.authn.security.ts` with authentication enforcement tests for every practice route. Per Pricing Gap Closure Spec B1-10.

**Test every authenticated practice route returns 401 without a valid session cookie:**

For each of these routes, send the request with:
1. No cookie at all
2. An expired session cookie
3. A tampered/invalid session cookie

All three variants must return HTTP 401.

**Routes to test:**

```typescript
const PROTECTED_ROUTES = [
  { method: 'POST',   path: '/api/v1/practices' },
  { method: 'GET',    path: '/api/v1/practices/:id' },
  { method: 'PATCH',  path: '/api/v1/practices/:id' },
  { method: 'GET',    path: '/api/v1/practices/:id/seats' },
  { method: 'POST',   path: '/api/v1/practices/:id/invitations' },
  { method: 'POST',   path: '/api/v1/practice-invitations/:token/accept' },
  { method: 'DELETE', path: '/api/v1/practices/:id/seats/:userId' },
  { method: 'GET',    path: '/api/v1/practices/:id/invoices' },
];
```

Replace `:id`, `:token`, `:userId` with valid-looking UUIDs/tokens for the test (the request should fail on auth before reaching validation).

**Test structure:**

```typescript
describe('Practice Authentication Security', () => {
  describe.each(PROTECTED_ROUTES)('$method $path', ({ method, path }) => {
    it('returns 401 with no session cookie', async () => {
      const res = await request(method, resolvedPath).send(validBody);
      expect(res.status).toBe(401);
    });

    it('returns 401 with expired session cookie', async () => {
      const res = await request(method, resolvedPath)
        .set('Cookie', expiredSessionCookie)
        .send(validBody);
      expect(res.status).toBe(401);
    });

    it('returns 401 with tampered session cookie', async () => {
      const res = await request(method, resolvedPath)
        .set('Cookie', 'session=tampered-invalid-value')
        .send(validBody);
      expect(res.status).toBe(401);
    });
  });
});
```

**Also verify:**
- 401 response bodies contain ONLY a generic error object (e.g., `{ error: 'Unauthorized' }`)
- 401 responses do NOT contain any practice data, member names, or internal details
- 401 responses do NOT vary based on whether the practice/resource exists (no information leakage via differing error messages)

## Critical Security Rules

- 401 responses must not contain any data beyond a generic error object.
- No practice names, member names, Stripe IDs, or any other internal data in 401 responses.
- Response format for 401 must be identical whether or not the practice/resource actually exists (prevents enumeration attacks).

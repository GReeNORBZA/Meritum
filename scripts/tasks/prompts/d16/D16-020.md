# Task D16-020: Service: createPractice — create practice, membership for admin, assign PRACTICE_ADMIN role, create Stripe customer

## What to Build

Add the `createPractice()` function to a new `apps/api/src/domains/platform/practice.service.ts` file (or add to the existing `platform.service.ts` — follow the pattern used in the codebase). Per Pricing Gap Closure Spec B1-7.

**Function signature:**
```typescript
async function createPractice(
  adminUserId: string,
  name: string,
  billingFrequency: 'MONTHLY' | 'ANNUAL',
): Promise<SelectPractice>
```

**Business logic:**

1. **Validate the admin user exists and is a physician:**
   - Query the users table for `adminUserId`
   - Verify the user has the PHYSICIAN role
   - If not a physician, throw a `BusinessRuleError` with code `NOT_A_PHYSICIAN`

2. **Check the user does not already admin a practice:**
   - Call `practiceRepo.findPracticeByAdminUserId(adminUserId)`
   - If found, throw `BusinessRuleError` with code `ALREADY_PRACTICE_ADMIN`

3. **Check the user is not already a member of another practice:**
   - Call `membershipRepo.findActiveMembershipByPhysicianId(adminUserId)`
   - If found, throw `BusinessRuleError` with code `ALREADY_ON_PRACTICE`

4. **Create the practice record:**
   - Set `admin_user_id` to `adminUserId`
   - Set `name` to the provided name
   - Set `billing_frequency` to the provided value
   - Set `status` to `'ACTIVE'`
   - Set `current_period_start` to now
   - Set `current_period_end` to now + 1 month (for MONTHLY) or now + 1 year (for ANNUAL)

5. **Create a practice membership for the admin:**
   - Call `membershipRepo.createMembership()` with:
     - `practice_id`: the new practice ID
     - `physician_user_id`: `adminUserId`
     - `billing_mode`: Determine based on admin's current subscription:
       - If admin has active early bird subscription: `'INDIVIDUAL_EARLY_BIRD'`
       - Otherwise: `'PRACTICE_CONSOLIDATED'`
     - `joined_at`: now

6. **Assign PRACTICE_ADMIN role to the user:**
   - Use the IAM service/repository to add the `PRACTICE_ADMIN` role to the user
   - The user retains their existing PHYSICIAN role (role stacking)

7. **Create a Stripe customer for the practice:**
   - Call `stripe.customers.create()` with:
     - `name`: practice name
     - `email`: admin user's email
     - `metadata.practice_id`: the practice ID
   - NO PHI sent to Stripe — only practice name and admin email
   - Update the practice record with the Stripe customer ID

8. **Audit log the practice creation:**
   - Log action: `'practice.created'`
   - Include: practice ID, admin user ID, practice name

9. **Return the created practice record.**

**Wrap steps 4-7 in a database transaction** to ensure atomicity. If any step fails, roll back.

## Critical Security Rules

- **ZERO PHI sent to Stripe.** Only practice name and admin email.
- The PRACTICE_ADMIN role grants ONLY practice management permissions (seat view/manage, invoice view, settings edit). It does NOT grant any CLAIM_*, PATIENT_*, ANALYTICS_*, or ADMIN_PHI_ACCESS permissions.
- Transaction ensures atomicity — partial practice creation is not allowed.

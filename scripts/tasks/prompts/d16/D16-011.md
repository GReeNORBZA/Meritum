# Task D16-011: Practice membership repository — add/remove members, billing mode queries, end-of-month removal queries

## What to Build

Create `apps/api/src/domains/platform/practice-membership.repository.ts` with all membership data access. Per Pricing Gap Closure Spec B1-2 and B1-7.

**Functions to implement:**

**createMembership(data: InsertPracticeMembership): Promise\<SelectPracticeMembership\>**
- Insert into `practice_memberships` table
- Return the created record

**findActiveMembershipByPhysicianId(physicianUserId: string): Promise\<SelectPracticeMembership | null\>**
- Find membership where `physician_user_id` matches AND `is_active = true`
- A physician can belong to at most ONE active practice

**findActiveMembershipsByPracticeId(practiceId: string): Promise\<SelectPracticeMembership[]\>**
- Return all active memberships for a practice (is_active = true)
- Used for seat listing and headcount

**findMembershipByPracticeAndPhysician(practiceId: string, physicianUserId: string): Promise\<SelectPracticeMembership | null\>**
- Find active membership matching both practice and physician
- Return null if not found or not active

**setRemovalScheduled(membershipId: string, removedAt: Date, removalEffectiveAt: Date): Promise\<void\>**
- Set `removed_at` and `removal_effective_at` on the membership
- Do NOT set `is_active = false` yet — that happens at end of month

**deactivateMembership(membershipId: string): Promise\<void\>**
- Set `is_active = false`
- Called by the end-of-month scheduled job

**findPendingRemovals(cutoffDate: Date): Promise\<SelectPracticeMembership[]\>**
- Find memberships where `removal_effective_at <= cutoffDate` AND `is_active = true`
- Used by `handleEndOfMonthRemovals()` scheduled job

**deactivateAllMemberships(practiceId: string): Promise\<void\>**
- Set `is_active = false` for ALL active memberships on the practice
- Used during practice dissolution (when headcount drops below 5)

**updateBillingMode(membershipId: string, billingMode: string): Promise\<void\>**
- Update the `billing_mode` column
- Used when an early bird physician transitions to PRACTICE_CONSOLIDATED

**findMembershipsByBillingMode(practiceId: string, billingMode: string): Promise\<SelectPracticeMembership[]\>**
- Return active memberships for a practice filtered by billing mode
- Used for Stripe quantity calculations and early bird transition handling

**countActiveMembersByBillingMode(practiceId: string, billingMode: string): Promise\<number\>**
- Count active memberships by billing mode
- Used for Stripe subscription quantity (only PRACTICE_CONSOLIDATED members)

## Critical Security Rules

- This repository queries ONLY the `practice_memberships` table. It NEVER joins with claims, patients, analytics, or any PHI-containing table.
- All queries use parameterized inputs via Drizzle ORM.
- The unique partial index on `physician_user_id WHERE is_active = true` enforces one-practice-per-physician at the database level.

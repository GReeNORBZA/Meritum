# Task D16-045: Security: audit — practice creation, invitation, acceptance, removal, dissolution all audit logged

## What to Build

Create `apps/api/test/security/platform/practice.audit.security.ts` with audit trail completeness tests. Per Pricing Gap Closure Spec B1-10.

**Test setup:**
- Create a practice, invite physicians, accept invitations, remove a physician, trigger dissolution
- After each action, verify the audit log contains the correct entry

## Critical Security Rules

- Every state-changing operation on a practice MUST be audit logged. There are no exceptions.
- Audit entries must include: action type, actor (who), target (what), timestamp, IP address.
- Audit logs must NOT contain the raw invitation token (security-sensitive).
- Audit logs must NOT contain PHI (patient names, UHN numbers, claim details).
- Audit entries for email invitations may contain the email address (this is necessary for the audit trail and is not PHI in this context).
- Audit logs must be append-only and tamper-resistant.

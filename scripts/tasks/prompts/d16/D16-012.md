# Task D16-012: Practice invitation repository — create, find by token, expire, accept/decline

## What to Build

Create `apps/api/src/domains/platform/practice-invitation.repository.ts` with all invitation data access. Per Pricing Gap Closure Spec B1-3.

**Functions to implement:**

**createInvitation(data: InsertPracticeInvitation): Promise\<SelectPracticeInvitation\>**
- Insert into `practice_invitations` table
- Return the created record

**findInvitationByTokenHash(tokenHash: string): Promise\<SelectPracticeInvitation | null\>**
- Find invitation by its hashed token
- Return null if not found

**findPendingInvitationByEmail(email: string, practiceId: string): Promise\<SelectPracticeInvitation | null\>**
- Find a PENDING invitation for the given email and practice
- Used to check if an invitation already exists

**findPendingInvitationsByPracticeId(practiceId: string): Promise\<SelectPracticeInvitation[]\>**
- Return all PENDING invitations for a practice

**updateInvitationStatus(invitationId: string, status: string): Promise\<void\>**
- Update the invitation status (ACCEPTED, DECLINED, EXPIRED)

**expireInvitations(cutoffDate: Date): Promise\<number\>**
- Set status to 'EXPIRED' for all PENDING invitations where `expires_at <= cutoffDate`
- Return the count of expired invitations
- Used by a scheduled job or called before token validation

**findInvitationsByEmail(email: string): Promise\<SelectPracticeInvitation[]\>**
- Find all invitations (any status) for an email
- Used for history/audit purposes

## Critical Security Rules

- The `token_hash` column stores SHA-256 hash of the invitation token. The raw token is NEVER stored in the database.
- Invitation lookup is by hash, not by raw token. The service layer hashes the incoming token before calling `findInvitationByTokenHash`.
- All queries use parameterized inputs via Drizzle ORM — no raw SQL string concatenation.
- Email addresses should be compared case-insensitively (use `.toLowerCase()` before querying or use SQL `LOWER()`).

# Task SCR-050: Add category column to hsc_codes schema and seed pipeline

## What to Build

Add the raw `category` field (e.g., "V Visit", "C Consultation", "14 Major Procedure") to the `hsc_codes` database table and propagate it through the seed pipeline to the API response.

### Why

The scraper extracts `category` from each HSC detail page and uses it to derive `feeType`, but the raw category is discarded. It contains richer information than the derived fee type and is useful for:
- Display in the UI (showing the SOMB category hierarchy)
- Filtering by category in search
- Debugging fee type mapping issues

### Changes Required

#### 1. Schema: `packages/shared/src/schemas/db/reference.schema.ts`

Add `category` column to the `hscCodes` table definition, after the `feeType` column:

```typescript
category: varchar('category', { length: 100 }),
```

No index needed — this is a display field, not a query filter.

#### 2. Seed: `apps/api/src/seed.ts`

In the HSC batch insert mapping (around line ~785), add:

```typescript
category: h.category ?? null,
```

The scraped JSON already has `category` on each HSC object — it just wasn't being mapped to the DB.

#### 3. Service: `apps/api/src/domains/reference/reference.service.ts`

Add `category` to the `HscDetailResult` interface:

```typescript
category: string | null;
```

Add to the `getHscDetail` mapping:

```typescript
category: code.category ?? null,
```

#### 4. Tests: `apps/api/src/domains/reference/reference.test.ts`

Update any test helpers that create mock HSC records to include `category`. Update test assertions for `getHscDetail` to verify `category` is returned.

#### 5. Migration

After updating the schema, generate and apply the migration:

```bash
cd apps/api && pnpm drizzle-kit generate && pnpm drizzle-kit migrate
```

**Note:** If the migration tools are not available in this environment, just create the schema change and note that migration generation is needed.

# Task SCR-051: Seed bundling exclusions into bundling_rules table

## What to Build

Wire the 135+ `bundlingExclusions` extracted by the enrichment script into the `bundling_rules` database table via the seed script.

### Why

The enrichment script (`enrich-hsc-data.ts`) extracts bundling exclusions from HSC notes text with fields `{ excludedCode, relationship }` and writes them to `hsc-codes.json`. But the seed script never reads these and never inserts them into the `bundling_rules` table. The validation check A19 (bundling warning) in Domain 4.1 depends on `bundling_rules` data.

### Changes Required

#### 1. Seed: `apps/api/src/seed.ts`

After the HSC codes and modifier eligibility inserts, add a new section to extract and insert bundling rules:

```typescript
// --- Bundling Rules from enriched HSC notes ---
console.log('  Seeding bundling rules from enriched HSC notes...');

const bundlingPairs = new Map<string, { codeA: string; codeB: string; relationship: string }>();

for (const hsc of scrapedHsc) {
  if (!hsc.bundlingExclusions?.length) continue;
  for (const excl of hsc.bundlingExclusions) {
    // Canonical ordering: codeA < codeB (matches bundling_rules table constraint)
    const [codeA, codeB] = [hsc.hscCode, excl.excludedCode].sort();
    const key = `${codeA}:${codeB}`;
    if (!bundlingPairs.has(key)) {
      bundlingPairs.set(key, {
        codeA,
        codeB,
        relationship: excl.relationship,
      });
    }
  }
}

const bundlingValues = Array.from(bundlingPairs.values());
const BUNDLING_BATCH_SIZE = 500;
for (let i = 0; i < bundlingValues.length; i += BUNDLING_BATCH_SIZE) {
  const batch = bundlingValues.slice(i, i + BUNDLING_BATCH_SIZE);
  await db.insert(bundlingRules).values(
    batch.map((b) => ({
      codeA: b.codeA,
      codeB: b.codeB,
      relationship: b.relationship,
      description: `Extracted from Fee Navigator notes: ${b.codeA} ${b.relationship.replace(/_/g, ' ')} ${b.codeB}`,
      sourceReference: 'Fee Navigator notes text',
      isActive: true,
    })),
  );
}
console.log(`    Bundling rules: ${bundlingValues.length} pairs inserted`);
```

#### 2. Import

Make sure `bundlingRules` is imported from the schema at the top of `seed.ts`:

```typescript
import { bundlingRules } from '@meritum/shared/schemas/db/reference.schema.js';
```

Check if it's already imported (it may be). If not, add it to the existing import statement.

#### 3. Type Safety

The `scrapedHsc` type loaded from JSON needs to include `bundlingExclusions`. Check the type assertion or interface used when loading `hsc-codes.json`. It should already have:
```typescript
bundlingExclusions?: Array<{ excludedCode: string; relationship: string }>;
```

If the type doesn't include this field, add it.

### Important

- The `bundling_rules` table has a unique index on `(code_a, code_b)`. The canonical ordering (`codeA < codeB`) and deduplication via Map ensures no constraint violations.
- Use `sort()` for canonical ordering — JavaScript string sort puts `01.01A` before `03.03A` which is correct.

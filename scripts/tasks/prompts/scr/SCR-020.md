# Task SCR-020: Replace hardcoded ROOT_SECTION_KEYS with dynamic discovery

## What to Build

Modify `scripts/scrape-fee-navigator.ts` to dynamically discover root section keys from the HSC main page instead of using a hardcoded array.

### Current Problem

`ROOT_SECTION_KEYS` is hardcoded to 19 numeric keys (line ~36). If the Fee Navigator adds or removes top-level sections, the scraper misses new codes or errors on removed sections.

### Solution

1. **Add a new function** `discoverRootSectionKeys()` that fetches the HSC main page and extracts root expandable node keys:

```typescript
async function discoverRootSectionKeys(): Promise<string[]> {
  console.log('  Discovering root section keys from HSC main page...');
  const html = await fetchWithRetry(`${BASE_URL}/hsc`);
  const $ = cheerio.load(html);

  const keys: string[] = [];
  $('div.node.expandable, a.node.expandable').each((_i, el) => {
    const key = $(el).attr('data-key');
    if (key && /^\d+$/.test(key)) {
      keys.push(key);
    }
  });

  if (keys.length === 0) {
    throw new Error(
      'No root section keys found on HSC main page — the site structure may have changed. ' +
      'Check https://apps.albertadoctors.org/fee-navigator/hsc manually.'
    );
  }

  console.log(`  Found ${keys.length} root section keys: [${keys.join(', ')}]`);
  return keys;
}
```

# Task SCR-052: Add billingTips and commonTerms columns to hsc_codes schema and seed

## What to Build

Add `billingTips` and `commonTerms` columns to the `hsc_codes` database table and propagate them through the seed pipeline to the API response.

### Why

The scraper extracts `billingTips` (string) and `commonTerms` (string array) from each HSC detail page, but neither is seeded into the database:
- `billingTips` contains practical billing guidance from the Fee Navigator — complements `helpText`
- `commonTerms` contains alternative names/phrases for the service — useful for search autocomplete

### Changes Required

#### 1. Schema: `packages/shared/src/schemas/db/reference.schema.ts`

Add two columns to the `hscCodes` table, after the `helpText` column:

```typescript
billingTips: text('billing_tips'),
commonTerms: jsonb('common_terms')
  .notNull()
  .default(sql`'[]'::jsonb`)
  .$type<string[]>(),
```

Optionally, add a GIN index on `commonTerms` for search (if the array values should be searchable):

```typescript
// In the table's index array:
index('hsc_codes_common_terms_gin_idx').using(
  'gin',
  sql`${table.commonTerms}`,
),
```

This uses the default JSONB GIN indexing which supports `@>` containment queries.

#### 2. Seed: `apps/api/src/seed.ts`

In the HSC batch insert mapping (around line ~785), add:

```typescript
billingTips: h.billingTips ?? null,
commonTerms: h.commonTerms ?? [],
```

The scraped JSON already has both fields — they just weren't mapped.

#### 3. Service: `apps/api/src/domains/reference/reference.service.ts`

Add to `HscDetailResult` interface:

```typescript
billingTips: string | null;
commonTerms: string[];
```

Add to the `getHscDetail` mapping:

```typescript
billingTips: code.billingTips ?? null,
commonTerms: (code.commonTerms as string[]) ?? [],
```

#### 4. Tests: `apps/api/src/domains/reference/reference.test.ts`

Update test helpers and assertions:
- Mock HSC records should include `billingTips` and `commonTerms` fields
- `getHscDetail` tests should verify these fields are returned
- For enrichment field tests (around line ~3228), add assertions for `billingTips` and `commonTerms`

#### 5. Migration

After updating the schema, migration needs to be generated:

```bash
cd apps/api && pnpm drizzle-kit generate && pnpm drizzle-kit migrate
```

**Note:** If migration tools are not available, just create the schema change.

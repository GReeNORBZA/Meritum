# Task SCR-060: Create post-scrape validation script

## What to Build

Create `scripts/validate-fee-navigator-data.ts` — a standalone validation script that reads the JSON output files in `scripts/data/fee-navigator/` and validates completeness, format, and cross-file consistency.

### Purpose

This script is the safety net for the entire pipeline. It catches:
- Partial scrapes (site went down mid-scrape)
- CAPTCHA blocks (scraper saved garbage data)
- Site structure changes (parsing produced empty/malformed records)
- Enrichment regressions (new regex broke extraction)
- Data duplication (resume bug)

### Validation Checks

The script should implement these categories of checks:

#### A. File existence
- All required files exist: `hsc-codes.json`, `hsc-modifiers.json`, `modifiers.json`, `governing-rules.json`, `explanatory-codes.json`, `scrape-metadata.json`

#### B. Completeness thresholds
- `hsc-codes.json`: record count ≥ 3,000
- `hsc-modifiers.json`: record count ≥ 40,000
- `modifiers.json`: record count ≥ 40
- `governing-rules.json`: record count ≥ 15
- `explanatory-codes.json`: record count ≥ 100

Log a warning (not failure) if counts drop by more than 5% from `scrape-metadata.json`.

#### C. HSC code format validation
For every record in `hsc-codes.json`:
- `hscCode` matches `/^\d{2}\.\d{2,3}[A-Z]{0,3}$/`
- `description` is a non-empty string with length > 3
- `feeType` is one of: `VISIT`, `PROCEDURE`, `FIXED`, `CONSULTATION`, `LABORATORY`, `RADIOLOGY`, `ANESTHESIA`, `THERAPEUTIC`, `OTHER`, `UNKNOWN`
- `baseFee` is null or matches `/^\d+\.\d{2}$/`
- `modifierEligibility` is an array of strings
- `governingRuleReferences` is an array of strings matching `/^\d+(\.\d+)*$/`
- `surchargeEligible` is boolean

#### D. No duplicates
- No duplicate `hscCode` values in `hsc-codes.json`

#### E. Modifier row validation
For every record in `hsc-modifiers.json`:
- `hscCode`, `type`, `code`, `action` are non-empty strings
- `amount` is a non-empty string

#### F. Cross-file consistency
- Every `hscCode` in `hsc-modifiers.json` exists in `hsc-codes.json`
- Every modifier `type` in `hsc-modifiers.json` exists in `modifiers.json` as a `modifierCode`

#### G. Enrichment validation (if fields present)
If any HSC record has enrichment fields:
- `requiresReferral` is boolean (not undefined)
- `specialtyRestrictions` is array (if present)
- `bundlingExclusions` is array of objects with `excludedCode` (string) and `relationship` (string) — if present
- `ageRestriction` is null or object with `text` (string) — if present
- `frequencyRestriction` is null or object with `text` (string), `count` (number), `period` (string) — if present
- `facilityDesignation` is null, `'in_office'`, or `'out_of_office'` — if present

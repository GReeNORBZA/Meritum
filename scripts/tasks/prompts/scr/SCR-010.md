# Task SCR-010: Fix baseFee comma stripping and surcharge detection in scraper

## What to Build

Fix two bugs in `scripts/scrape-fee-navigator.ts` in the `parseHscDetailHtml` function.

### Bug 1: baseFee comma stripping (line ~351)

**Current code:**
```typescript
baseFee = feeMatch ? feeMatch[1].replace(',', '') : null;
```

**Problem:** `.replace(',', '')` only removes the first comma. For fees ≥ $1,000,000 (unlikely but possible), this produces incorrect values.

**Fix:**
```typescript
baseFee = feeMatch ? feeMatch[1].replace(/,/g, '') : null;
```

### Bug 2: surchargeEligible detection (line ~435-437)

**Current code:**
```typescript
if (type === 'SURC' || type === 'SURT' || modCode.includes('SURC')) {
  surchargeEligible = true;
}
```

**Problem:** Checks `modCode.includes('SURC')` for sub-codes but doesn't check for SURT sub-codes.

**Fix:**
```typescript
if (type === 'SURC' || type === 'SURT' || modCode.includes('SURC') || modCode.includes('SURT')) {
  surchargeEligible = true;
}
```

### Both fixes are in `parseHscDetailHtml` function only.

Make sure the scraper still imports shared utilities from `scripts/lib/fee-navigator-utils.ts` (done in SCR-001).

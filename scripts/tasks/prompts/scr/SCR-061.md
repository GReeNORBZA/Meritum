# Task SCR-061: Integrate validation into scraper and enrichment pipelines

## What to Build

Wire the validation script (`scripts/validate-fee-navigator-data.ts`, created in SCR-060) into the end of both the scraper and enrichment scripts so validation runs automatically.

### Changes

#### 1. Scraper: `scripts/scrape-fee-navigator.ts`

At the end of the `main()` function, after saving metadata and cleaning up the progress file, add:

```typescript
// Run post-scrape validation
console.log('\n=== Running post-scrape validation ===\n');
try {
  const { execSync } = await import('node:child_process');
  const tsxPath = path.join(process.cwd(), 'apps', 'api', 'node_modules', '.bin', 'tsx');
  const validateScript = path.join(path.dirname(new URL(import.meta.url).pathname), 'validate-fee-navigator-data.ts');
  execSync(`"${tsxPath}" "${validateScript}"`, { stdio: 'inherit' });
  console.log('\n  Post-scrape validation: PASSED\n');
} catch {
  console.error('\n  *** POST-SCRAPE VALIDATION FAILED ***');
  console.error('  Review the validation output above before using this data.');
  console.error('  The scraped data may be incomplete or corrupted.\n');
  process.exit(2); // Exit code 2 = validation failure (distinct from 1 = scrape error)
}
```

Place this BEFORE the final summary log (the `=== Scrape Complete! ===` section), so the summary includes the validation result.

#### 2. Enrichment: `scripts/enrich-hsc-data.ts`

At the end of the `main()` function, after the verification section, add the same validation call:

```typescript
// Run post-enrichment validation
console.log('\n=== Running post-enrichment validation ===\n');
try {
  const { execSync } = await import('node:child_process');
  const tsxPath = path.join(process.cwd(), 'apps', 'api', 'node_modules', '.bin', 'tsx');
  const validateScript = path.join(
    path.dirname(new URL(import.meta.url).pathname),
    'validate-fee-navigator-data.ts',
  );
  execSync(`"${tsxPath}" "${validateScript}"`, { stdio: 'inherit' });
  console.log('\n  Post-enrichment validation: PASSED\n');
} catch {
  console.error('\n  *** POST-ENRICHMENT VALIDATION FAILED ***');
  console.error('  Review the validation output above before using this data.');
  console.error('  The enrichment may have corrupted the data.\n');
  process.exit(2);
}
```

### Important Notes

- The `execSync` import uses dynamic `await import()` because the scripts are ESM modules.
- Use `stdio: 'inherit'` so the validation output streams directly to the console.
- Exit code 2 distinguishes validation failure from scrape/enrichment errors (exit code 1).
- The `tsxPath` resolution uses the API workspace's tsx binary (same as the task runner). If `tsx` is available globally via `npx`, use `execSync('npx tsx ...')` instead. Check which approach works in this project.

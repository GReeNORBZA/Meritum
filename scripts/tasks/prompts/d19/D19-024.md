# Task D19-024: Tests: PCPCM payment recording, reconciliation logic, panel size update, physician scoping

## What to Build

Create comprehensive tests for the PCPCM payment reconciliation feature. Add tests to `apps/api/src/domains/provider/provider.test.ts` (new `describe` block) or a dedicated `apps/api/src/domains/provider/pcpcm-payment.test.ts`.

### Test Setup

Create test fixtures:
- **Provider A**: Has an active PCPCM enrolment with panel_size = 500.
- **Provider B**: Has an active PCPCM enrolment with panel_size = 300. (For cross-tenant isolation testing.)
- **Provider C**: No PCPCM enrolment (FFS only). Used to test rejection.

### Test Scenarios

```typescript
describe('PCPCM Payment Recording', () => {
  it('records EXPECTED payment with only expectedAmount');
  it('records RECEIVED payment with only actualAmount');
  it('records RECEIVED payment with both expectedAmount and actualAmount');
  it('stores panelSizeAtPayment when provided');
  it('stores notes when provided');
  it('rejects payment recording for provider without PCPCM enrolment');
  it('rejects payment with invalid enrolmentId');
  it('rejects payment where enrolmentId belongs to different provider');
  it('validates paymentPeriodStart is before paymentPeriodEnd');
  it('audit logs each payment recording');
});

describe('PCPCM Reconciliation Logic', () => {
  describe('Happy Path', () => {
    it('reconciles payment when expected matches actual exactly');
    it('reconciles payment when difference is within $0.01 tolerance');
    it('sets status to RECONCILED for matching payments');
    it('sets reconciledAt timestamp on reconciled payments');
    it('returns correct reconciled count');
  });

  describe('Discrepancy Detection', () => {
    it('marks DISCREPANCY when actual exceeds expected by more than $0.01');
    it('marks DISCREPANCY when actual is less than expected by more than $0.01');
    it('includes discrepancy amount in notes');
    it('returns correct discrepancy count');
    it('sets reconciledAt even for discrepancies');
  });

  describe('Skip Logic', () => {
    it('skips payments with only expectedAmount (no actual yet)');
    it('skips payments with only actualAmount (no expected yet)');
    it('skips already-reconciled payments');
    it('skips already-discrepancy payments');
  });

  describe('Multi-Period', () => {
    it('reconciles payments across multiple periods in one run');
    it('handles mix of reconciled and discrepancy in same run');
    it('returns details for each processed payment');
  });

  it('audit logs the reconciliation run with counts');
});

describe('PCPCM Payment History', () => {
  it('returns all payments for provider ordered by period_end DESC');
  it('filters by status EXPECTED');
  it('filters by status RECEIVED');
  it('filters by status RECONCILED');
  it('filters by status DISCREPANCY');
  it('filters by period date range');
  it('paginates with page and pageSize');
  it('returns total count for pagination');
  it('returns hasMore=true when more pages exist');
  it('returns empty list for provider with no payments');
});

describe('Panel Size Update', () => {
  it('updates panel_size on pcpcm_enrolments table');
  it('rejects panel size of 0');
  it('rejects negative panel size');
  it('rejects non-integer panel size');
  it('rejects if enrolment does not belong to provider');
  it('rejects if provider has no active PCPCM enrolment');
  it('audit logs the update with old and new panel size');
  it('emits PCPCM_PANEL_SIZE_UPDATED event');
});

describe('PCPCM Physician Scoping', () => {
  it('Provider A cannot see Provider B payments');
  it('Provider B cannot see Provider A payments');
  it('Provider A reconciliation does not affect Provider B payments');
  it('Provider A panel size update does not affect Provider B enrolment');
  it('listPcpcmPaymentsForProvider returns ZERO records for wrong providerId');
  it('findPcpcmPaymentById returns undefined for correct paymentId but wrong providerId');
  it('updatePanelSize rejects enrolmentId belonging to different provider');
});

describe('PCPCM Route Security', () => {
  it('GET /providers/me/pcpcm/payments returns 401 without auth');
  it('POST /providers/me/pcpcm/payments returns 401 without auth');
  it('POST /providers/me/pcpcm/reconcile returns 401 without auth');
  it('PATCH /providers/me/pcpcm/panel-size returns 401 without auth');
  it('non-PHYSICIAN role gets 403 on all PCPCM routes');
  it('DELEGATE role with appropriate permissions can access PCPCM routes');
});
```

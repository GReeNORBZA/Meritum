# Task D19-002: Service: handleAnnualCancellation() — 6-month forfeit rule, prorated refund calculation

## What to Build

Add the `handleAnnualCancellation()` function to `apps/api/src/domains/platform/platform.service.ts`.

**Function signature:**

```typescript
export async function handleAnnualCancellation(
  deps: PlatformServiceDeps,
  userId: string,
  subscriptionId: string,
  eventEmitter?: PlatformEventEmitter,
): Promise<AnnualCancellationResult>
```

**Return type:**

```typescript
export interface AnnualCancellationResult {
  policy: CancellationPolicy;
  message: string;
  refund: {
    refundAmount: number;
    monthsUsed: number;
    monthsRemaining: number;
    monthlyRate: number;
  } | null;
  cancellationEffectiveAt: Date;
}
```

**Implementation logic (per spec B4-1):**

1. Look up the subscription by `subscriptionId` using `deps.subscriptionRepo.findSubscriptionByProviderId(userId)`.
2. Validate the subscription exists, belongs to the user, and is an annual plan (`plan` contains `'ANNUAL'`). If not annual, throw `BusinessRuleError('handleAnnualCancellation is only for annual plans')`.
3. Calculate `monthsElapsed` = `Math.floor((now - currentPeriodStart) / (30.44 * 24 * 60 * 60 * 1000))`. Use 30.44 days as average month length for consistency.
4. Call `determineCancellationPolicy(subscription.plan, monthsElapsed)` from the constants.
5. **If FORFEIT_PERIOD (monthsElapsed < 6):**
   - No refund. Subscription continues until `currentPeriodEnd`.
   - Schedule cancellation at period end via Stripe: call `deps.stripe.subscriptions.update(stripeSubscriptionId, { cancel_at_period_end: true })`. Note: you may need to add `update` to the `StripeClient` interface's `subscriptions` property.
   - Return message using `ANNUAL_CANCELLATION_FORFEIT_MESSAGE` with `[period end date]` replaced by actual date.
   - Emit `CANCELLATION_SCHEDULED` event.
   - Audit log with action `'annual_cancellation.forfeit'`.
6. **If PRORATED_REFUND (monthsElapsed >= 6):**
   - Call `calculateAnnualRefund(annualAmount, monthsElapsed)` to get refund details.
   - The `annualAmount` comes from `SubscriptionPlanPricing[subscription.plan].amount` parsed as a number.
   - Do NOT issue the Stripe refund here (that is D19-003). Instead, return the calculated refund details.
   - Schedule cancellation at period end.
   - Emit `CANCELLATION_WITH_REFUND_PENDING` event with refund amount.
   - Audit log with action `'annual_cancellation.prorated_refund'`.

**Add to the StripeClient interface:**

```typescript
subscriptions: {
  cancel(subscriptionId: string): Promise<{ id: string; status: string }>;
  update(subscriptionId: string, params: {
    cancel_at_period_end?: boolean;
  }): Promise<{ id: string; status: string; cancel_at_period_end: boolean }>;
};
```

## Critical Security Rules

- Verify the subscription belongs to the requesting user before processing.
- No PHI in Stripe API calls or event payloads.
- Refund amounts must be calculated server-side; never trust client-provided amounts.
- Audit log every cancellation request with the policy applied.

# Task D19-005: Tests: annual cancellation — 6-month forfeit, prorated refund, monthly cancellation passthrough

## What to Build

Create comprehensive test coverage for the annual cancellation and refund flow in `apps/api/src/domains/platform/platform.test.ts` (add a new `describe` block) or a dedicated test file `apps/api/src/domains/platform/cancellation.test.ts`.

**Test scenarios to cover:**

### Annual cancellation — Forfeit period (months 1-5)

```typescript
describe('Annual Cancellation — Forfeit Period', () => {
  it('month 1: no refund, subscription continues until period end');
  it('month 3: no refund, message includes period end date');
  it('month 5 (boundary): no refund, returns FORFEIT_PERIOD policy');
  it('forfeit: calls stripe.subscriptions.update with cancel_at_period_end: true');
  it('forfeit: does NOT call stripe.refunds.create');
  it('forfeit: emits CANCELLATION_SCHEDULED event');
  it('forfeit: audit log action is annual_cancellation.forfeit');
  it('forfeit: response message contains the actual period end date');
});
```

### Annual cancellation — Prorated refund (months 6-12)

```typescript
describe('Annual Cancellation — Prorated Refund', () => {
  it('month 6 (boundary): refund for 6 remaining months');
  it('month 8: refund for 4 remaining months');
  it('month 11: refund for 1 remaining month');
  it('month 12: refund is $0 (no remaining months)');

  // Exact amount verification using formula: refund = (12 - months_used) * (annual_amount / 12)
  it('STANDARD_ANNUAL at month 6: refund = 6 * ($3181 / 12) = $1590.50');
  it('STANDARD_ANNUAL at month 9: refund = 3 * ($3181 / 12) = $795.25');
  it('CLINIC_ANNUAL at month 7: refund = 5 * ($2863 / 12) = $1192.92');
  it('EARLY_BIRD_ANNUAL at month 10: refund = 2 * ($2388 / 12) = $398.00');

  it('prorated: calls stripe.refunds.create with correct amount in cents');
  it('prorated: records REFUNDED payment in payment_history');
  it('prorated: refund payment has negative amount values');
  it('prorated: emits REFUND_PROCESSED event');
  it('prorated: emits CANCELLATION_WITH_REFUND_PENDING event');
  it('prorated: audit log action is annual_cancellation.prorated_refund');
  it('prorated: GST is included in refund calculation (5% of base refund)');
});
```

### Monthly cancellation (no change needed — passthrough)

```typescript
describe('Monthly Cancellation — Passthrough', () => {
  it('STANDARD_MONTHLY: returns MONTHLY_CANCEL policy');
  it('EARLY_BIRD_MONTHLY: returns MONTHLY_CANCEL policy');
  it('CLINIC_MONTHLY: returns MONTHLY_CANCEL policy');
  it('monthly: calls stripe.subscriptions.update with cancel_at_period_end: true');
  it('monthly: does NOT call stripe.refunds.create');
  it('monthly: does NOT call handleAnnualCancellation');
  it('monthly: cancellation effective at currentPeriodEnd');
  it('monthly: audit log action is subscription.monthly_cancellation');
});
```

### Edge cases

```typescript
describe('Cancellation Edge Cases', () => {
  it('rejects cancellation for non-existent subscription (404)');
  it('rejects cancellation for already-cancelled subscription');
  it('rejects handleAnnualCancellation for monthly plan');
  it('handles subscription exactly at 6-month boundary correctly');
  it('handles leap year date calculations gracefully');
});
```

### Security

```typescript
describe('Cancellation Security', () => {
  it('returns 401 without authentication');
  it('physician A cannot cancel physician B subscription');
  it('cancellation response does not leak Stripe payment_intent IDs');
  it('audit log captures userId for every cancellation request');

```

**Test setup:**

- Mock the Stripe client (`stripe.subscriptions.update`, `stripe.refunds.create`).
- Mock the subscription repository with test subscription records at various `currentPeriodStart` dates.
- Mock the payment repository with PAID payment records for refund lookups.
- Use `vi.useFakeTimers()` or fixed dates to control "now" for month calculations.

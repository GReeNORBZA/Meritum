# Task D19-023: Routes: PCPCM payment endpoints — GET/POST payments, POST reconcile, PATCH panel-size

## What to Build

Add PCPCM payment routes to `apps/api/src/domains/provider/provider.routes.ts`.

**New routes:**

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | /api/v1/providers/me/pcpcm/payments | PHYSICIAN | List PCPCM payment history |
| POST | /api/v1/providers/me/pcpcm/payments | PHYSICIAN | Record a PCPCM payment |
| POST | /api/v1/providers/me/pcpcm/reconcile | PHYSICIAN | Trigger payment reconciliation |
| PATCH | /api/v1/providers/me/pcpcm/panel-size | PHYSICIAN | Update panel size (manual entry) |

### Zod Validation Schemas

Add to `packages/shared/src/schemas/provider.schema.ts` (or wherever provider Zod schemas are defined):

```typescript
import { z } from 'zod';

export const createPcpcmPaymentSchema = z.object({
  enrolmentId: z.string().uuid(),
  paymentPeriodStart: z.string().regex(/^\d{4}-\d{2}-\d{2}$/, 'Must be YYYY-MM-DD format'),
  paymentPeriodEnd: z.string().regex(/^\d{4}-\d{2}-\d{2}$/, 'Must be YYYY-MM-DD format'),
  expectedAmount: z.number().positive().optional(),
  actualAmount: z.number().positive().optional(),
  panelSizeAtPayment: z.number().int().positive().optional(),
  notes: z.string().max(1000).optional(),
}).refine(
  (data) => data.expectedAmount !== undefined || data.actualAmount !== undefined,
  { message: 'At least one of expectedAmount or actualAmount must be provided' },
);

export const listPcpcmPaymentsQuerySchema = z.object({
  status: z.enum(['EXPECTED', 'RECEIVED', 'RECONCILED', 'DISCREPANCY']).optional(),
  periodStart: z.string().regex(/^\d{4}-\d{2}-\d{2}$/).optional(),
  periodEnd: z.string().regex(/^\d{4}-\d{2}-\d{2}$/).optional(),
  page: z.coerce.number().int().positive().default(1),
  pageSize: z.coerce.number().int().positive().max(100).default(20),
});

export const updatePanelSizeSchema = z.object({
  enrolmentId: z.string().uuid(),
  panelSize: z.number().int().positive('Panel size must be a positive integer'),
});
```

### Handler Implementation

```typescript
// GET /api/v1/providers/me/pcpcm/payments
async function listPcpcmPaymentsHandler(request, reply) {
  const providerId = request.authContext.providerId;
  const filters = request.query; // validated by Zod schema

  const result = await getPcpcmPaymentHistory(deps, providerId, {
    status: filters.status,
    periodStart: filters.periodStart,
    periodEnd: filters.periodEnd,
    page: filters.page,
    pageSize: filters.pageSize,
  });

  return reply.code(200).send({ data: result.data, pagination: result.pagination });
}

// POST /api/v1/providers/me/pcpcm/payments
async function createPcpcmPaymentHandler(request, reply) {
  const providerId = request.authContext.providerId;
  const actorId = request.authContext.userId;
  const body = request.body; // validated by Zod schema

  const payment = await recordPcpcmPayment(deps, providerId, body, actorId);
  return reply.code(201).send({ data: payment });
}

// POST /api/v1/providers/me/pcpcm/reconcile
async function reconcilePcpcmPaymentsHandler(request, reply) {
  const providerId = request.authContext.providerId;
  const actorId = request.authContext.userId;

  const result = await reconcilePcpcmPayments(deps, providerId, actorId);
  return reply.code(200).send({ data: result });
}

// PATCH /api/v1/providers/me/pcpcm/panel-size
async function updatePanelSizeHandler(request, reply) {
  const providerId = request.authContext.providerId;
  const actorId = request.authContext.userId;
  const { enrolmentId, panelSize } = request.body; // validated by Zod schema

  await updatePanelSize(deps, providerId, enrolmentId, panelSize, actorId);
  return reply.code(200).send({ data: { success: true, panelSize } });
}
```

### Route Registration

Add to the existing `providerRoutes` function in `provider.routes.ts`:

```typescript
// =========================================================================
// PCPCM Payment Routes
// =========================================================================

app.get('/api/v1/providers/me/pcpcm/payments', {
  schema: { querystring: listPcpcmPaymentsQuerySchema },
  preHandler: [app.authenticate, requireRole('PHYSICIAN')],
  handler: listPcpcmPaymentsHandler,
});

app.post('/api/v1/providers/me/pcpcm/payments', {
  schema: { body: createPcpcmPaymentSchema },
  preHandler: [app.authenticate, requireRole('PHYSICIAN')],
  handler: createPcpcmPaymentHandler,
});

app.post('/api/v1/providers/me/pcpcm/reconcile', {
  preHandler: [app.authenticate, requireRole('PHYSICIAN')],
  handler: reconcilePcpcmPaymentsHandler,
});

app.patch('/api/v1/providers/me/pcpcm/panel-size', {
  schema: { body: updatePanelSizeSchema },
  preHandler: [app.authenticate, requireRole('PHYSICIAN')],
  handler: updatePanelSizeHandler,
});
```

## Critical Security Rules

- All routes require PHYSICIAN role authentication.
- `providerId` is derived from `request.authContext.providerId` — never from URL parameters or request body.
- A physician can only access their own PCPCM payment data.
- Input validation via Zod schemas prevents injection attacks.
- All mutations are audit logged.

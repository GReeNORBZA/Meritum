# Task D19-003: Service: Stripe refund integration — refundAnnualSubscription(), payment_history REFUNDED recording, REFUND_PROCESSED event

## What to Build

Add the `refundAnnualSubscription()` function and Stripe Refunds API integration to `apps/api/src/domains/platform/platform.service.ts`.

**Extend the StripeClient interface** to include the refunds API:

```typescript
refunds: {
  create(params: {
    payment_intent?: string;
    charge?: string;
    amount: number;
    reason?: 'duplicate' | 'fraudulent' | 'requested_by_customer';
    metadata?: Record<string, string>;
  }): Promise<{
    id: string;
    amount: number;
    status: string;
    currency: string;
  }>;
};
```

**Function signature:**

```typescript
export async function refundAnnualSubscription(
  deps: PlatformServiceDeps,
  subscriptionId: string,
  refundAmountCad: number,
  monthsUsed: number,
  eventEmitter?: PlatformEventEmitter,
): Promise<RefundResult>
```

**Return type:**

```typescript
export interface RefundResult {
  stripeRefundId: string;
  refundAmountCad: number;
  gstRefundAmount: number;
  totalRefundCad: number;
  paymentHistoryId: string;
}
```

**Implementation logic:**

1. Look up the subscription by `subscriptionId` using `deps.subscriptionRepo.findSubscriptionById(subscriptionId)`. You may need to add this method to the repository interface if it does not exist — it should find by `subscription_id` PK.
2. Find the most recent PAID payment for this subscription from `payment_history` using `deps.paymentRepo.findLatestPaidPayment(subscriptionId)`. You may need to add this method. It returns the record with the `stripe_invoice_id` needed for the refund.
3. Calculate GST on the refund: `gstRefundAmount = parseFloat((refundAmountCad * GST_RATE).toFixed(2))`.
4. Calculate total refund: `totalRefundCad = parseFloat((refundAmountCad + gstRefundAmount).toFixed(2))`.
5. Convert to cents for Stripe: `amountCents = Math.round(totalRefundCad * 100)`.
6. Call the Stripe Refunds API:
   ```typescript
   const stripeRefund = await deps.stripe.refunds.create({
     payment_intent: latestPayment.stripePaymentIntentId, // or charge
     amount: amountCents,
     reason: 'requested_by_customer',
     metadata: {
       meritum_subscription_id: subscriptionId,
       months_used: String(monthsUsed),
       refund_type: 'annual_prorated',
     },
   });
   ```
7. Record the refund in `payment_history` with status `PaymentStatus.REFUNDED`:
   ```typescript
   const refundPayment = await deps.paymentRepo.recordPayment({
     subscriptionId,
     stripeInvoiceId: `refund_${stripeRefund.id}`,
     amountCad: `-${refundAmountCad.toFixed(2)}`,
     gstAmount: `-${gstRefundAmount.toFixed(2)}`,
     totalCad: `-${totalRefundCad.toFixed(2)}`,
     status: PaymentStatus.REFUNDED,
     paidAt: new Date(),
   });
   ```
8. Emit `REFUND_PROCESSED` event:
   ```typescript
   eventEmitter?.emit('REFUND_PROCESSED', {
     subscriptionId,
     stripeRefundId: stripeRefund.id,
     refundAmountCad: totalRefundCad,
     monthsUsed,
   });
   ```
9. Audit log with action `'subscription.refund_processed'`.

**Repository additions needed:**

If not already present, add to `PaymentRepository` interface:
- `findLatestPaidPayment(subscriptionId: string): Promise<SelectPaymentHistory | undefined>` — finds most recent payment with status PAID for the given subscription, ordered by `created_at DESC`.

If not already present, add to `SubscriptionRepository` interface:
- `findSubscriptionById(subscriptionId: string): Promise<SelectSubscription | undefined>` — finds by PK.

## Critical Security Rules

- Refund amounts are calculated server-side from `calculateAnnualRefund()`. Never accept client-provided refund amounts.
- Stripe secret key used only via the injected client. Never logged.
- No PHI in Stripe refund metadata. Only subscription_id and months_used.
- Payment history records negative amounts for refunds to maintain accurate ledger.
- Audit log every refund with full traceability.

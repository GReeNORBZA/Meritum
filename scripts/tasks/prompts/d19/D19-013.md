# Task D19-013: Integration tests: full portability export — ZIP contents, physician scoping, cancelled-status access

## What to Build

Create integration tests for the full data portability export at `apps/api/test/integration/platform/data-portability.integration.ts`.

These tests verify the complete flow from request to ZIP generation, using realistic test data across all 6 CSV export types.

### Test Setup

Create test fixtures:
- **Physician A**: Active subscription with AHCIP claims, WCB claims, patients, assessments, analytics data, and intelligence suggestion events.
- **Physician B**: Active subscription with their own separate data (for cross-tenant isolation testing).
- **Physician C**: Cancelled subscription within the 30-day grace period (`deletionScheduledAt` in the future).
- **Physician D**: Cancelled subscription past the 30-day grace period (`deletionScheduledAt` in the past).

### Test Scenarios

```typescript
describe('Full Portability Export — Integration', () => {
  describe('ZIP Contents Verification', () => {
    it('ZIP contains exactly 6 files: ahcip_claims.csv, wcb_claims.csv, patients.csv, assessments.csv, analytics_summary.csv, intelligence_history.csv');
    it('each CSV file has a valid header row');
    it('ahcip_claims.csv contains all AHCIP claims for the physician');
    it('wcb_claims.csv contains all WCB claims with child records');
    it('patients.csv contains all patient records for the physician');
    it('assessments.csv contains assessment results, rejections, and batch outcomes');
    it('analytics_summary.csv contains dashboard metrics and report history');
    it('intelligence_history.csv contains suggestion events with actions');
    it('ZIP filename follows format: meritum_export_{physicianId}_{date}.zip');
    it('ZIP is a valid archive that can be extracted');
  });

  describe('Physician Scoping — No Cross-Tenant Data', () => {
    it('Physician A export contains ZERO records belonging to Physician B');
    it('Physician B export contains ZERO records belonging to Physician A');
    it('AHCIP claims CSV only contains claims where provider_id = physicianId');
    it('WCB claims CSV only contains claims where provider_id = physicianId');
    it('patients CSV only contains patients where provider_id = physicianId');
    it('assessments CSV only contains assessments for physicianId claims');
    it('analytics CSV only contains metrics for physicianId');
    it('intelligence CSV only contains suggestions for physicianId');
  });

  describe('Cancelled Status — 30-Day Grace Window', () => {
    it('Physician C (cancelled, within grace): export succeeds');
    it('Physician C: ZIP contains all 6 CSVs with correct data');
    it('Physician D (cancelled, past grace): export fails with BusinessRuleError');
    it('SUSPENDED physician with DATA_EXPORT feature: export succeeds');
    it('ACTIVE physician: export always succeeds');
  });

  describe('Edge Cases', () => {
    it('physician with no WCB claims: wcb_claims.csv has header only');
    it('physician with no intelligence events: intelligence_history.csv has header only');
    it('physician with no analytics data: analytics_summary.csv has header only');
    it('handles large data sets without timeout (performance)');
  });

  describe('Security', () => {
    it('returns 401 without authentication');
    it('returns 403 for non-PHYSICIAN role');
    it('physician A cannot request export for physician B data');
    it('export request is audit logged');
  });
});
```

### Test Approach

- Use the Fastify `inject` method to call the export endpoint.
- Parse the returned ZIP buffer using a library like `adm-zip` or `yauzl` to verify contents.
- For each CSV, parse and count rows to verify correct physician scoping.
- Seed test data using the test database helpers used elsewhere in the project.

## Critical Security Rules

- The most critical assertion: **no cross-tenant data leakage**. Every CSV in the ZIP must contain ONLY data for the requesting physician.
- Test with at least 2 physicians to verify isolation.
- Verify the 30-day grace window is enforced (not 29 days, not 31 days).

# Task D19-021: Repository: PCPCM payment CRUD, physician-scoped queries, reconciliation status queries

## What to Build

Add PCPCM payment repository functions to `apps/api/src/domains/provider/provider.repository.ts` (or create a separate `pcpcm-payment.repository.ts` file within the provider domain).

**Repository interface to implement:**

```typescript
export interface PcpcmPaymentRepository {
  // --- CRUD ---
  createPcpcmPayment(data: InsertPcpcmPayment): Promise<SelectPcpcmPayment>;
  findPcpcmPaymentById(paymentId: string, providerId: string): Promise<SelectPcpcmPayment | undefined>;
  updatePcpcmPayment(
    paymentId: string,
    providerId: string,
    data: Partial<InsertPcpcmPayment>,
  ): Promise<SelectPcpcmPayment | undefined>;

  // --- Physician-Scoped Queries ---
  listPcpcmPaymentsForProvider(
    providerId: string,
    filters?: {
      status?: string;
      periodStart?: string;
      periodEnd?: string;
      limit?: number;
      offset?: number;
    },
  ): Promise<{ data: SelectPcpcmPayment[]; total: number }>;

  // --- Reconciliation Queries ---
  findUnreconciledPayments(providerId: string): Promise<SelectPcpcmPayment[]>;
  findPaymentsForPeriod(
    providerId: string,
    periodStart: string,
    periodEnd: string,
  ): Promise<SelectPcpcmPayment[]>;
  updatePaymentStatus(
    paymentId: string,
    providerId: string,
    status: string,
    reconciledAt?: Date,
    notes?: string,
  ): Promise<SelectPcpcmPayment | undefined>;

  // --- Panel Size ---
  updatePanelSizeOnEnrolment(
    enrolmentId: string,
    providerId: string,
    panelSize: number,
  ): Promise<void>;
}
```

**Implementation details for each method:**

### `createPcpcmPayment(data)`
- Insert into `pcpcm_payments` table.
- `data.providerId` is required. Validate it matches an existing provider.
- Default `status` to `'EXPECTED'` if not provided.
- Return the created record.

### `findPcpcmPaymentById(paymentId, providerId)`
- Select from `pcpcm_payments` WHERE `payment_id = paymentId AND provider_id = providerId`.
- **Always scope by providerId** to enforce tenant isolation.

### `updatePcpcmPayment(paymentId, providerId, data)`
- Update the record WHERE `payment_id = paymentId AND provider_id = providerId`.
- Return the updated record or undefined if not found.

### `listPcpcmPaymentsForProvider(providerId, filters)`
- Select from `pcpcm_payments` WHERE `provider_id = providerId`.
- Apply optional filters: `status`, `payment_period_start >= periodStart`, `payment_period_end <= periodEnd`.
- Order by `payment_period_end DESC`.
- Support pagination with `limit` and `offset`.
- Return `{ data, total }` for paginated results.

### `findUnreconciledPayments(providerId)`
- Select WHERE `provider_id = providerId AND status IN ('EXPECTED', 'RECEIVED')`.
- Order by `payment_period_start ASC`.

### `findPaymentsForPeriod(providerId, periodStart, periodEnd)`
- Select WHERE `provider_id = providerId AND payment_period_start >= periodStart AND payment_period_end <= periodEnd`.

### `updatePaymentStatus(paymentId, providerId, status, reconciledAt?, notes?)`
- Update `status`, optionally `reconciled_at` and `notes`.
- WHERE `payment_id = paymentId AND provider_id = providerId`.

### `updatePanelSizeOnEnrolment(enrolmentId, providerId, panelSize)`
- Update the `panel_size` column on the existing `pcpcm_enrolments` table.
- WHERE `enrolment_id = enrolmentId AND provider_id = providerId`.

**Use Drizzle ORM** for all queries, following the pattern established in the existing `provider.repository.ts`.

## Critical Security Rules

- **Every query MUST include `provider_id` in the WHERE clause.** This is the physician-scoped tenant isolation boundary. No exceptions.
- Never return data for a different physician.
- Use parameterized queries (Drizzle handles this automatically).

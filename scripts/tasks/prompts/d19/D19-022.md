# Task D19-022: Service: recordPcpcmPayment(), reconcilePcpcmPayments(), getPcpcmPaymentHistory(), updatePanelSize()

## What to Build

Add PCPCM payment service functions to `apps/api/src/domains/provider/provider.service.ts` (or a new `pcpcm-payment.service.ts` file within the provider domain).

**Extend the ProviderServiceDeps** (or create a new deps interface) to include:

```typescript
pcpcmPaymentRepo: PcpcmPaymentRepository;
```

### Function 1: `recordPcpcmPayment()`

```typescript
export async function recordPcpcmPayment(
  deps: ProviderServiceDeps,
  providerId: string,
  paymentData: {
    enrolmentId: string;
    paymentPeriodStart: string; // ISO date
    paymentPeriodEnd: string;   // ISO date
    expectedAmount?: number;
    actualAmount?: number;
    panelSizeAtPayment?: number;
    notes?: string;
  },
  actorId: string,
): Promise<SelectPcpcmPayment>
```

**Logic:**
1. Validate the provider has an active PCPCM enrolment matching `enrolmentId`.
2. Determine initial status:
   - If only `expectedAmount` provided: status = `'EXPECTED'`
   - If only `actualAmount` provided: status = `'RECEIVED'`
   - If both provided: status = `'RECEIVED'` (will be reconciled in next step)
3. Create the payment record via `pcpcmPaymentRepo.createPcpcmPayment()`.
4. Audit log with action `'pcpcm_payment.recorded'`.
5. Return the created record.

### Function 2: `reconcilePcpcmPayments()`

```typescript
export async function reconcilePcpcmPayments(
  deps: ProviderServiceDeps,
  providerId: string,
  actorId: string,
): Promise<ReconciliationResult>
```

**Return type:**

```typescript
export interface ReconciliationResult {
  reconciled: number;
  discrepancies: number;
  details: Array<{
    paymentId: string;
    periodStart: string;
    periodEnd: string;
    expectedAmount: number | null;
    actualAmount: number | null;
    status: string;
    discrepancyAmount?: number;
  }>;
}
```

**Logic:**
1. Get all unreconciled payments for the provider (`status IN ('EXPECTED', 'RECEIVED')`).
2. For each payment where BOTH `expectedAmount` AND `actualAmount` are set:
   - Compare expected vs actual.
   - If they match (within a tolerance of $0.01): set status = `'RECONCILED'`, set `reconciledAt = now()`.
   - If they don't match: set status = `'DISCREPANCY'`, set `reconciledAt = now()`, add note with the difference.
3. Payments with only `expectedAmount` or only `actualAmount` are skipped (not ready for reconciliation).
4. Audit log with action `'pcpcm_payment.reconciliation_run'` including counts.
5. Return reconciliation summary.

### Function 3: `getPcpcmPaymentHistory()`

```typescript
export async function getPcpcmPaymentHistory(
  deps: ProviderServiceDeps,
  providerId: string,
  filters?: {
    status?: string;
    periodStart?: string;
    periodEnd?: string;
    page?: number;
    pageSize?: number;
  },
): Promise<{
  data: SelectPcpcmPayment[];
  pagination: { total: number; page: number; pageSize: number; hasMore: boolean };
}>
```

**Logic:**
1. Call `pcpcmPaymentRepo.listPcpcmPaymentsForProvider(providerId, filters)`.
2. Map pagination parameters (`page` and `pageSize` to `limit` and `offset`).
3. Return paginated results.

### Function 4: `updatePanelSize()`

```typescript
export async function updatePanelSize(
  deps: ProviderServiceDeps,
  providerId: string,
  enrolmentId: string,
  panelSize: number,
  actorId: string,
): Promise<void>
```

**Logic:**
1. Validate `panelSize` is a positive integer.
2. Validate the enrolment belongs to the provider and is active.
3. Call `pcpcmPaymentRepo.updatePanelSizeOnEnrolment(enrolmentId, providerId, panelSize)`.
4. Audit log with action `'pcpcm_panel_size.updated'` including old and new panel size.
5. Emit `'PCPCM_PANEL_SIZE_UPDATED'` event.

**Note:** Per spec B4-4, "MVP: physician enters the number from their AH quarterly panel attribution report." This is a simple manual entry — no estimation logic.

## Critical Security Rules

- All operations are scoped by `providerId`. No cross-physician data access.
- Panel size is entered manually by the physician from their AH report. No automated scraping.
- Audit log every payment recording, reconciliation, and panel size update.
- Reconciliation discrepancy amounts are logged but not automatically reported externally.

# Task D19-004: Routes: cancellation endpoint with monthly vs annual differentiation

## What to Build

Add a cancellation route to the platform routes file at `apps/api/src/domains/platform/platform.routes.ts` (or the equivalent routes registration file for the platform domain).

**New route:**

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| POST | /api/v1/subscriptions/cancel | PHYSICIAN | Request cancellation of current subscription |

**Handler logic:**

```typescript
// POST /api/v1/subscriptions/cancel
async function cancelSubscriptionHandler(request, reply) {
  const userId = request.authContext.userId;

  // 1. Look up current subscription
  const subscription = await subscriptionRepo.findSubscriptionByProviderId(userId);
  if (!subscription) {
    return reply.code(404).send({
      error: { code: 'NOT_FOUND', message: 'No active subscription found' },
    });
  }

  // 2. Determine cancellation type based on plan
  const isAnnual = subscription.plan.includes('ANNUAL');

  if (isAnnual) {
    // Use handleAnnualCancellation for annual plans
    const result = await handleAnnualCancellation(deps, userId, subscription.subscriptionId, eventEmitter);

    // If prorated refund is applicable, also issue the refund
    if (result.policy === CancellationPolicy.PRORATED_REFUND && result.refund) {
      const refundResult = await refundAnnualSubscription(
        deps,
        subscription.subscriptionId,
        result.refund.refundAmount,
        result.refund.monthsUsed,
        eventEmitter,
      );

      return reply.code(200).send({
        data: {
          policy: result.policy,
          message: result.message,
          refund: {
            amount: refundResult.refundAmountCad,
            gst: refundResult.gstRefundAmount,
            total: refundResult.totalRefundCad,
            stripeRefundId: refundResult.stripeRefundId,
          },
          cancellationEffectiveAt: result.cancellationEffectiveAt.toISOString(),
        },
      });
    }

    // Forfeit period — no refund
    return reply.code(200).send({
      data: {
        policy: result.policy,
        message: result.message,
        refund: null,
        cancellationEffectiveAt: result.cancellationEffectiveAt.toISOString(),
      },
    });
  }

  // 3. Monthly plan: cancel at period end via Stripe
  await deps.stripe.subscriptions.update(subscription.stripeSubscriptionId, {
    cancel_at_period_end: true,
  });

  eventEmitter?.emit('CANCELLATION_SCHEDULED', {
    subscriptionId: subscription.subscriptionId,
    providerId: userId,
    plan: subscription.plan,
    effectiveAt: subscription.currentPeriodEnd.toISOString(),
  });

  await deps.auditLogger?.log({
    action: 'subscription.monthly_cancellation',
    resourceType: 'subscription',
    resourceId: subscription.subscriptionId,
    actorType: 'user',
    metadata: { userId, plan: subscription.plan },
  });

  return reply.code(200).send({
    data: {
      policy: CancellationPolicy.MONTHLY_CANCEL,
      message: 'Your subscription will be cancelled at the end of the current billing period.',
      refund: null,
      cancellationEffectiveAt: subscription.currentPeriodEnd.toISOString(),
    },
  });
}
```

## Critical Security Rules

- Endpoint requires PHYSICIAN role authentication.
- A physician can only cancel their own subscription (enforced by using `request.authContext.userId`).
- No cross-user cancellation is possible.
- All cancellation requests are audit logged.
- Refund amounts are server-calculated, never client-supplied.

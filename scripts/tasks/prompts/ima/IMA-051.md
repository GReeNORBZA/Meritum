# IMA-051: Complete HI export — service + handler + route

## What to Build

### 1. Service Function

- `generateFullHiExport(ctx: AuthContext, format: 'csv' | 'json')`:
  - Call repo to get all HI
  - Generate one CSV (or JSON) file per entity type
  - Bundle into a ZIP archive with a `manifest.json` containing:
    - `export_date`, `provider_id`, `format`, `entity_counts` (per type), `schema_version`
  - Write ZIP to DO Spaces under path `exports/{provider_id}/{timestamp}.zip`
  - Generate presigned download URL (72h expiry for full exports)
  - Create a `generatedReports` record with type `FULL_DATA_PORTABILITY`
  - Emit `FULL_HI_EXPORT_READY` notification
  - Emit audit `export.full_hi_requested` on start, `export.full_hi_ready` on completion
  - Return `{ reportId, downloadUrl, expiresAt }`

### 2. Handler

- `generateFullExport` — parse fullHiExportSchema, call service, return 202 (async processing). If the dataset is small (< 10MB estimated), can return 200 with URL directly.

### 3. Route

- `POST /api/v1/platform/export/full` with `preHandler: [authenticate, authorize('DATA_EXPORT')]`, `config: { auditLog: true }`.

This endpoint must remain accessible in CANCELLED subscription state (feature access already preserved per existing platform constants).

## Critical Security Rules

- Presigned URL scoped to physician.
- The download URL expires after 72 hours.
- Export runs as a background job for large datasets.
- Audit entries must NOT contain PHI.

## Prerequisites

- IMA-050 (repository)

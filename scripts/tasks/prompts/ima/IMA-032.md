# IMA-032: Breach notification ‚Äî handlers + routes (admin-only)

## What to Build

Create breach handlers (extend platform handlers):

- `createBreach` ‚Äî admin only, parse createBreachSchema
- `getBreach` ‚Äî admin only, return breach with updates and notification status
- `listBreaches` ‚Äî admin only, paginated
- `sendNotifications` ‚Äî admin only, trigger notification send for a breach
- `addUpdate` ‚Äî admin only, add supplementary update
- `resolveBreach` ‚Äî admin only, resolve breach

Register routes in platform.routes.ts:

- `POST /api/v1/platform/breaches` ‚Äî admin only
- `GET /api/v1/platform/breaches` ‚Äî admin only
- `GET /api/v1/platform/breaches/:id` ‚Äî admin only
- `POST /api/v1/platform/breaches/:id/notify` ‚Äî admin only
- `POST /api/v1/platform/breaches/:id/updates` ‚Äî admin only
- `POST /api/v1/platform/breaches/:id/resolve` ‚Äî admin only

All breach endpoints require admin role. No physician-facing breach endpoints (they receive notifications via the notification system, not through breach-specific API endpoints).

## Critical Security Rules

- ALL endpoints admin-only.
- No PHI in any breach response body (describe breach in general terms).
- Breach records contain sensitive operational data ‚Äî never expose to non-admin roles.

## Prerequisites

- IMA-031 (service)

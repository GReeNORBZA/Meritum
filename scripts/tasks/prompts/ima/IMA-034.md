# IMA-034: Breach notification — security tests

## What to Build

Create `apps/api/test/security/platform/platform.breach.security.ts`:

### Authentication

All 6 breach endpoints return 401 without session:
- POST /api/v1/platform/breaches
- GET /api/v1/platform/breaches
- GET /api/v1/platform/breaches/:id
- POST /api/v1/platform/breaches/:id/notify
- POST /api/v1/platform/breaches/:id/updates
- POST /api/v1/platform/breaches/:id/resolve

### Authorization

All 6 breach endpoints return 403 for physician role. All return 403 for delegate role.

### Input Validation

- SQL injection in breach_description
- XSS in breach_description
- Non-UUID breach ID
- Invalid status values
- Empty affected_provider_ids array
- Negative affected_count

### Leakage Prevention

- Error responses contain no PHI
- 404 for non-existent breach (not 500)
- No stack traces in error responses

### Audit Trail

- Breach creation produces audit record
- Notification send produces audit record
- Update produces audit record
- Resolution produces audit record

## Critical Security Rules

- Every breach endpoint must enforce admin-only access.
- No PHI may appear in any error response body.
- Input validation must reject all malformed or malicious input before processing.

## Prerequisites

- IMA-033 (integration tests)

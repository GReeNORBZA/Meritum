# IMA-060: Data destruction — extend pipeline with DO Spaces cleanup, backup tracking, confirmation email

## What to Build

Read the existing `apps/api/src/domains/platform/platform.service.ts` and find
the `runDeletionCheck()` function (the scheduled job that executes PHI deletion
after the grace period).

Extend this function and the surrounding infrastructure:

### 1. DO Spaces file cleanup

After deleting records from active database tables, delete all files in DO
Spaces scoped to the provider:

- Delete all objects under the prefix `exports/{provider_id}/`
- Delete all objects under the prefix `reports/{provider_id}/`
- Delete all objects under the prefix `uploads/{provider_id}/`
- Use the S3-compatible API (already configured via `SPACES_ENDPOINT`,
  `SPACES_ACCESS_KEY`, `SPACES_SECRET_KEY` env vars).
- If the S3 client does not exist yet, create a utility function in
  `apps/api/src/lib/spaces.ts` that wraps the AWS SDK S3 client
  (S3-compatible).

### 2. Data destruction tracking

After each stage of deletion, create or update a `data_destruction_tracking`
record:

- After DB deletion: set `activeDeletedAt = now()`
- After Spaces deletion: set `filesDeletedAt = now()`
- Set `backupPurgeDeadline = activeDeletedAt + 90 days`
- Backup purge is a manual ops process. The system tracks the deadline and
  alerts admin.

### 3. Confirmation email

Create a new scheduled job `runDestructionConfirmation()` that runs daily:

- Find tracking records where `backupPurgedAt` is set but `confirmationSentAt`
  is null.
- Send `DATA_DESTRUCTION_CONFIRMED` notification to the physician's last known
  email.
- Set `confirmationSentAt`.
- Emit audit `destruction.confirmed`.
- Also find records where `backupPurgeDeadline` has passed but
  `backupPurgedAt` is null — alert admin (log warning or internal
  notification).

### 4. Admin marking backup purge

Add a simple admin endpoint or service function:

- `markBackupPurged(adminCtx, providerId)` — sets `backupPurgedAt = now()`.
  Admin calls this after confirming backups are purged.
- Emit audit `destruction.backup_purged`.

## Critical Security Rules

- The confirmation email goes to the LAST KNOWN email (stored before account
  deactivation). Do not attempt to look up the email from the deactivated user
  record if it has been cleared.
- Store the email in the tracking record or in a field that survives
  deactivation.

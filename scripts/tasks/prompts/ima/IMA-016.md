# IMA-016: Secondary email -- handler, route, integration tests, notification dual-delivery

## What to Build

1. Extend `apps/api/src/domains/iam/iam.handlers.ts` with:
   - `updateSecondaryEmail` handler: parse body with
     `updateSecondaryEmailSchema`, call service, return
     `{ data: { secondary_email } }`

2. Extend `apps/api/src/domains/iam/iam.routes.ts`:
   - `PUT /api/v1/account/secondary-email` with
     `preHandler: [authenticate]`, body schema `updateSecondaryEmailSchema`

3. Extend the notification service
   (`apps/api/src/domains/notification/notification.service.ts`):
   - In the email delivery function: for notifications with event types
     `BREACH_INITIAL_NOTIFICATION`, `BREACH_UPDATE`, `IMA_AMENDMENT_NOTICE`,
     `IMA_AMENDMENT_REMINDER`, `IMA_AMENDMENT_DEADLINE`, also send to the
     recipient's secondary email if set.
   - Implementation: after sending to primary email, check if the physician has
     a secondary email; if so, queue a second email delivery to that address.

4. Create integration test file
   `apps/api/test/integration/iam/secondary-email.integration.ts`.

## Critical Security Rules

- PUT (not PATCH) because this is a full replacement of the secondary email
  value.
- 401 without session.
- The secondary email is not exposed in any public-facing response (only in
  `GET /api/v1/account` for the authenticated user).

## Prerequisites

- IMA-015 (repository + service)

## Tests to Write

Create `apps/api/test/integration/iam/secondary-email.integration.ts`:

```typescript
it('PUT /api/v1/account/secondary-email sets the secondary email');
it('PUT /api/v1/account/secondary-email with null clears it');
it('PUT /api/v1/account/secondary-email rejects same as primary');
it('PUT /api/v1/account/secondary-email returns 401 without session');
it('GET /api/v1/account includes secondary_email in response');
it('breach notification is delivered to both primary and secondary email');
```

## Run After Completion

```bash
pnpm --filter api vitest run test/integration/iam/account.integration.ts
```

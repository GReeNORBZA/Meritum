# IMA-050: Complete HI export — repository

## What to Build

Create a comprehensive data compilation repository function. This can live in the platform repository or a new `apps/api/src/domains/platform/export.repository.ts`.

- `getCompleteHealthInformation(providerId: string)` — queries ALL Health Information for a physician across every PHI table:
  - patients (all, including inactive)
  - claims + claim_line_items + claim_validations (all states)
  - assessments + assessment_line_items
  - wcb_claims + wcb_form_data (if tables exist)
  - batches + batch_claims
  - pcpcm_enrolments + pcpcm_payments
  - provider profile + business_arrangements + practice_locations
  - delegate_relationships (names and permissions)
  - submission_preferences
  - ai_suggestions (all statuses)
  - audit_log entries for this provider
  - ed_shifts + favourite_codes

Every query MUST include `WHERE provider_id = :providerId`. Return data structured by entity type.

For tables that don't exist yet in the codebase, return empty arrays with TODO comments.

## Critical Security Rules

- This is a complete dump of a physician's PHI. Tenant isolation is paramount — triple-check that every query is scoped.
- This function should ONLY be callable by the authenticated physician themselves (or admin for data hold scenarios).

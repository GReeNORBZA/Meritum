# IMA-012: Export window notifications

## What to Build

Wire the 4 new export window notification events into the platform
dunning/cancellation flow.

Read the existing `apps/api/src/domains/platform/platform.service.ts` to
understand the dunning sequence (`runDunningCheck`, `runCancellationCheck`,
`runDeletionCheck`).

Modify the flow:

1. In `runCancellationCheck()` (or wherever subscription transitions to
   CANCELLED): after setting `deletionScheduledAt`, emit
   `EXPORT_WINDOW_STARTED` notification to the physician. The export window is
   now 45 days (changed in IMA-001).

2. Create a new scheduled function `runExportWindowReminders()` that runs daily
   and:
   - Finds subscriptions where `status = 'CANCELLED'` and
     `deletionScheduledAt` is within the next 15 days
   - At 15 days remaining: emit `EXPORT_WINDOW_REMINDER`
   - At 7 days remaining: emit `EXPORT_WINDOW_CLOSING` (URGENT priority)
   - At 1 day remaining: emit `EXPORT_WINDOW_CLOSING` (URGENT priority,
     different template)
   - When `deletionScheduledAt` is reached: emit `EXPORT_WINDOW_CLOSED` before
     triggering deletion

3. Add notification templates for these 4 events in the notification constants
   or seeder.

## Prerequisites

- IMA-001 (notification event types)
- IMA-005 (migration applied)

## Tests to Write

Create or extend `apps/api/test/integration/platform/` test files:

```typescript
it('cancellation emits EXPORT_WINDOW_STARTED notification');
it('export window reminder emitted at 15 days remaining');
it('export window closing emitted at 7 days remaining');
it('EXPORT_WINDOW_CLOSED emitted when deletion begins');
it('export window uses 45-day period, not 30');
```

# IMA-041: Patient access request export — service + handler + route

## What to Build

### 1. Service Function

In the patient service (or a new export service):

- `exportPatientHealthInformation(ctx, patientId)` — calls repo to get all HI, formats as CSV files (one per entity type), bundles into a ZIP, writes to DO Spaces (or temp storage), generates a presigned download URL (24h expiry). Emits `PATIENT_ACCESS_EXPORT_READY` notification. Emits audit `export.patient_access_requested`. Returns `{ exportId, downloadUrl, expiresAt }`.

### 2. Handler

- `exportPatientHi` — parse patientExportParamSchema, call service, return 202 with exportId (async generation) or 200 with downloadUrl (sync for small datasets).

### 3. Route

- `POST /api/v1/patients/:id/export` with `preHandler: [authenticate, authorize('DATA_EXPORT')]`, `config: { auditLog: true }`.

If ZIP generation infrastructure doesn't exist yet, use Node.js `archiver` package or built-in zlib. Write CSV using simple string concatenation (no heavy library needed for structured data).

## Critical Security Rules

- Presigned URL must be scoped to the requesting physician.
- The download URL expires after 24h.
- Export audit entry must NOT contain the patient's PHI — only the patient_id and provider_id.

## Prerequisites

- IMA-040 (repository)

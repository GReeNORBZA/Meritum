# IMA-011: Patient correction audit trail

## What to Build

Extend the patient update flow to support formal corrections per IMA S3.10.

1. Read the existing patient handler at
   `apps/api/src/domains/patient/patient.handlers.ts` to understand the current
   PATCH flow.

2. Add a new route `PATCH /api/v1/patients/:id/correct` that:
   - Uses the `patientCorrectionSchema` from
     `packages/shared/src/schemas/compliance.schema.ts`
   - Requires `correction_reason` (mandatory)
   - Fetches the current patient record (old values)
   - Computes a field-level diff (for each changed field:
     `{ field, old_value, new_value }`)
   - Applies the update through the existing patient repository
   - Writes a structured audit log entry with action
     `patient.correction_applied`:

     ```typescript
     {
       action: 'patient.correction_applied',
       resourceType: 'patient',
       resourceId: patientId,
       detail: {
         correction_reason: req.body.correction_reason,
         changes: [
           { field: 'first_name', old_value: 'Jon', new_value: 'John' },
           { field: 'date_of_birth', old_value: '1990-01-01', new_value: '1990-01-02' },
         ]
       }
     }
     ```

3. Register the route in `apps/api/src/domains/patient/patient.routes.ts` with
   `preHandler: [authenticate, authorize('PATIENT_EDIT')]`.

## Critical Security Rules

- The diff must be computed server-side by comparing DB values to submitted
  values. Never trust client-provided "old values."
- PHN masking in audit detail: if PHN changes, old_value should be masked
  (e.g., `"123******"`).
- Tenant isolation: the patient must belong to the authenticated physician
  (provider_id check in repository).

## Prerequisites

- IMA-001 (audit action constant)
- IMA-004 (Zod schema)

## Tests to Write

Create or extend `apps/api/test/integration/patient/` test files:

```typescript
it('PATCH /api/v1/patients/:id/correct updates patient and records correction audit');
it('correction audit entry contains field-level diff with old and new values');
it('correction_reason is required');
it('at least one field besides correction_reason must be provided');
it('PHN is masked in correction audit detail');
it('returns 404 when correcting another physician\'s patient');
```

## Run After Completion

```bash
pnpm --filter api vitest run test/integration/patient/
```

# IMA-042: Patient access request export — integration + security tests

## What to Build

### Integration Tests

Create `apps/api/test/integration/patient/export.integration.ts`:

```typescript
describe('Patient Access Request Export (HIA Part 2)', () => {
  it('physician exports their patient -> receives ZIP with CSV files');
  it('export includes patient demographics');
  it('export includes claims linked to patient (if any)');
  it('export download URL is time-limited');
  it('PATIENT_ACCESS_EXPORT_READY notification sent');
  it('audit log records the export');
});
```

### Security Tests

Add security tests to existing `apps/api/test/security/patient/` files:

**Authentication:**
- POST /patients/:id/export returns 401 without session.

**Scoping:**
- Physician cannot export another physician's patient (returns 404).

**Input Validation:**
- Non-UUID patient ID returns 400.

**Leakage Prevention:**
- Error response for non-existent patient reveals no details.

**Audit:**
- Export produces audit record.

## Critical Security Rules

- Verify tenant isolation in every integration test scenario.
- Confirm that no PHI appears in audit log entries for export operations.
- Ensure presigned URLs cannot be accessed by other physicians.

## Prerequisites

- IMA-041 (service + handler + route)

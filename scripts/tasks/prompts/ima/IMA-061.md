# IMA-061: Data destruction — integration tests

## What to Build

Create `apps/api/test/integration/platform/destruction.integration.ts`:

```typescript
describe('Data Destruction Pipeline (IMA §8.4)', () => {
  // Setup: create physician, add patients and claims (if tables exist),
  // create reports, simulate subscription cancellation

  describe('Deletion execution', () => {
    it('after 45-day grace period, PHI is deleted from active tables');
    it('patients table has no records for the provider');
    it('audit log entries are preserved but PII is stripped');
    it('IMA records are preserved (legal evidence)');
    it('DO Spaces files for the provider are deleted');
    it('destruction tracking record created with correct timestamps');
  });

  describe('Backup purge tracking', () => {
    it('backupPurgeDeadline is 90 days after active deletion');
    it('admin can mark backup as purged');
    it('marking backup purged is admin-only (403 for physician)');
  });

  describe('Destruction confirmation', () => {
    it('after backup purge marked, confirmation email sent to last known email');
    it('confirmationSentAt set after email sent');
    it('confirmation job is idempotent');
  });

  describe('Overdue tracking', () => {
    it('admin alerted when backup purge deadline passes without purge');
  });
});
```

## Prerequisites

- IMA-060 (extended pipeline with DO Spaces cleanup, backup tracking,
  confirmation email)

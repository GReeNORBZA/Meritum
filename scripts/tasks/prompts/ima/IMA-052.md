# IMA-052: Complete HI export — integration + security tests

## What to Build

Create `apps/api/test/integration/platform/export.integration.ts`:

```typescript
describe('Complete Health Information Export (IMA §8.3)', () => {
  // Setup: physician with patients, claims (if claim tables exist), provider profile

  it('physician requests full export → 202 accepted');
  it('export ZIP contains patients.csv with all patient records');
  it('export ZIP contains provider_profile.csv');
  it('export ZIP contains manifest.json with entity counts');
  it('download URL is time-limited (72h)');
  it('FULL_HI_EXPORT_READY notification sent');
  it('export accessible in CANCELLED subscription state');
  it('export accessible in SUSPENDED subscription state');
  it('delegate without DATA_EXPORT permission cannot export (403)');
});
```

Add security tests (can extend existing platform security test files or create
`apps/api/test/security/platform/export.security.ts`):

- **authn:** `POST /platform/export/full` returns 401 without session.
- **authz:** Delegate without `DATA_EXPORT` permission gets 403.
- **scoping:** Export contains ONLY the authenticated physician's data. Create
  two physicians with data; verify export for physician1 contains zero records
  from physician2.
- **input:** Invalid `format` parameter rejected.
- **leakage:** Error responses contain no PHI.
- **audit:** Export request produces an audit record. Export completion produces
  an audit record.

## Prerequisites

- IMA-051 (service + handler + route)

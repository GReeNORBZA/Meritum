# IMA-010: PHI read-access audit logging across all domains

## What to Build

The current audit log plugin in `apps/api/src/plugins/auth.plugin.ts` only logs
state-changing methods (POST/PUT/PATCH/DELETE). IMA S4.3(d) requires logging
access to Health Information, which includes GET requests.

The mechanism already exists: route config `auditLog: true` overrides the
default. The task is to add this config to every GET route that returns PHI.

Modify the following route files to add `config: { auditLog: true }` to all
applicable GET routes:

- `apps/api/src/domains/patient/patient.routes.ts` -- all GET routes
- `apps/api/src/domains/provider/provider.routes.ts` -- all GET routes that
  return PHI (profile, BAs, locations, delegates, WCB config, preferences)
- `apps/api/src/domains/iam/iam.routes.ts` -- `GET /api/v1/account` (returns
  user profile with email)

Do NOT add audit logging to:

- Reference data GETs (public fee schedule data, not PHI)
- Health check endpoints
- Notification feed GETs (contain references, not PHI directly)

Example of how to add it to an existing route:

```typescript
app.get('/api/v1/patients', {
  config: { auditLog: true },  // <-- add this
  preHandler: [authenticate, authorize('PATIENT_VIEW')],
  handler: patientHandlers.list,
});
```

## Critical Security Rules

- Only add `auditLog: true` to routes that return Health Information (patient
  data, claim data, provider PHI).
- Do not add to reference data, notifications, or public endpoints.

## Prerequisites

None (modifying existing route files).

## Tests to Write

Create or extend `apps/api/test/security/iam/iam.audit.security.ts`:

```typescript
it('GET /api/v1/patients produces an audit log entry');
it('GET /api/v1/patients/:id produces an audit log entry');
it('GET /api/v1/providers/profile produces an audit log entry');
it('GET /api/v1/account produces an audit log entry');
it('GET /api/v1/reference/hsc-codes does NOT produce an audit log entry');
```

## Run After Completion

```bash
pnpm --filter api vitest run test/security/iam/iam.audit.security.ts
```

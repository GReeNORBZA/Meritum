# IMA-015: Secondary email -- repository + service + unit tests

## What to Build

1. Extend `apps/api/src/domains/iam/iam.repository.ts` with:
   - `updateSecondaryEmail(userId: string, email: string | null)` -- set or
     clear `secondary_email`
   - `getSecondaryEmail(userId: string)` -- return `secondary_email` value

2. Extend `apps/api/src/domains/iam/iam.service.ts` with:
   - `updateSecondaryEmail(ctx: AuthContext, email: string | null)`:
     - If email is not null: validate format, verify it is different from the
       user's primary email
     - Update via repository
     - Emit `account.secondary_email_updated` audit event with old and new
       values (mask if needed)
     - If email is null: clearing secondary email

## Critical Security Rules

- Secondary email cannot equal primary email (enforce in service).
- Audit log the change (old value to new value).
- The secondary email is used for breach and IMA amendment notifications only --
  it is NOT an alternative login.

## Prerequisites

- IMA-002 (secondary_email column on users)
- IMA-005 (migration)

## Tests to Write

Create or extend `apps/api/src/domains/iam/iam.test.ts`:

```typescript
it('updateSecondaryEmail sets the secondary email');
it('updateSecondaryEmail rejects email that matches primary');
it('updateSecondaryEmail with null clears the field');
it('updateSecondaryEmail emits audit event');
it('getSecondaryEmail returns the stored value');
```

## Run After Completion

```bash
pnpm --filter api vitest run src/domains/iam/iam.test.ts
```

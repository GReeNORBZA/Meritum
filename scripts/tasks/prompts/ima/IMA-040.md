# IMA-040: Patient access request export — repository

## What to Build

Extend `apps/api/src/domains/patient/patient.repository.ts` with:

- `getPatientHealthInformation(ctx: AuthContext, patientId: string)` — returns ALL Health Information for a specific patient scoped to the authenticated physician:
  - Patient demographics (from patients table)
  - All claims linked to this patient (from claims table, all states)
  - All claim line items for those claims
  - All assessments and assessment line items linked to those claims
  - All WCB claims and form data linked to this patient (if WCB tables exist)
  - Audit log entries referencing this patient (resourceType='patient', resourceId=patientId)

The query MUST include `WHERE provider_id = ctx.providerId` on every table join (tenant isolation). Return structured data grouped by entity type.

If some tables (claims, wcb_claims) don't exist yet in the codebase, return empty arrays for those entity types with a TODO comment. The export structure should accommodate them.

## Critical Security Rules

- Physician tenant isolation on EVERY query. Never return another physician's data even if the patient_id is known.
- Return 404 if the patient doesn't belong to the authenticated physician.

## Prerequisites

- IMA-005 (migration)

## Tests to Write

```typescript
it('getPatientHealthInformation returns patient demographics');
it('getPatientHealthInformation returns all claims for that patient');
it('getPatientHealthInformation scoped to authenticated physician only');
it('getPatientHealthInformation returns 404 for another physician\'s patient');
it('getPatientHealthInformation returns empty claims array when patient has no claims');
```

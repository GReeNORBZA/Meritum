# IMA-033: Breach notification — integration tests

## What to Build

Create `apps/api/test/integration/platform/breach.integration.ts`:

```typescript
describe('Breach Notification Lifecycle', () => {
  // Setup: admin user, two physicians with active subscriptions and secondary emails

  describe('Breach creation', () => {
    it('admin creates breach affecting both physicians -> 201');
    it('breach record includes evidence_hold_until = awareness_date + 12 months');
    it('affected custodians linked to breach');
  });

  describe('Notification send', () => {
    it('admin triggers notify -> sends email to both physicians');
    it('physician with secondary_email receives two emails');
    it('physician without secondary_email receives one email');
    it('custodians marked as notified after send');
    it('second notify call is idempotent (no duplicate emails)');
  });

  describe('Supplementary updates', () => {
    it('admin posts update -> all affected custodians receive BREACH_UPDATE');
    it('update stored in breach_updates (append-only)');
  });

  describe('Resolution', () => {
    it('admin resolves breach -> status RESOLVED, resolvedAt set');
  });

  describe('72-hour deadline tracking', () => {
    it('overdue breach detected when 72h passed with unnotified custodians');
  });
});
```

## Critical Security Rules

- Tests must verify that breach notifications contain no PHI.
- Tests must verify append-only behavior of breach_updates (no mutation of existing records).

## Prerequisites

- IMA-032 (handlers + routes)

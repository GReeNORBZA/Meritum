# IMA-030: Breach notification — repository

## What to Build

Extend `apps/api/src/domains/platform/platform.repository.ts` (or a companion file) with breach repository functions:

- `createBreachRecord(data)` — insert into breach_records. Set evidenceHoldUntil to MAX(awarenessDate + 12 months). Return breach record.
- `findBreachById(breachId)` — return breach with affected custodian count and update history.
- `listBreaches(filters)` — paginated, filterable by status.
- `updateBreachStatus(breachId, status, resolvedAt?)` — update status. If RESOLVED, set resolvedAt.
- `addAffectedCustodian(breachId, providerId)` — insert into breach_affected_custodians.
- `markCustodianNotified(breachId, providerId, method)` — set initialNotifiedAt and method.
- `getUnnotifiedCustodians(breachId)` — return affected custodians where initialNotifiedAt is null.
- `createBreachUpdate(breachId, data)` — insert into breach_updates (append-only).
- `listBreachUpdates(breachId)` — return all updates ordered by sentAt.
- `getOverdueBreaches()` — return breaches where status != RESOLVED and awareness_date + 72h < now() and there exist unnotified custodians.

## Critical Security Rules

- `breach_updates` is append-only — no update or delete functions may be implemented for this table.
- `evidenceHoldUntil` is computed server-side, never from client input.

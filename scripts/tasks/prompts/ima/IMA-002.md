# Task IMA-002: Drizzle schema — secondary_email on users, IMA amendment tables

## What to Build

### 1. Modify `packages/shared/src/schemas/db/iam.schema.ts`:

Add a `secondaryEmail` column to the `users` table:
```typescript
secondaryEmail: varchar('secondary_email', { length: 100 }),
```
Place it after the existing `email` field. Nullable — not all users will set one.

### 2. Create or extend `packages/shared/src/schemas/db/platform.schema.ts`:

Add two new tables at the end of the file:

**ima_amendments table:**
```typescript
export const imaAmendments = pgTable('ima_amendments', {
  amendmentId: uuid('amendment_id').primaryKey().defaultRandom(),
  amendmentType: varchar('amendment_type', { length: 20 }).notNull(),  // NON_MATERIAL or MATERIAL
  title: text('title').notNull(),
  description: text('description').notNull(),
  documentHash: varchar('document_hash', { length: 64 }).notNull(),   // SHA-256
  noticeDate: timestamp('notice_date', { withTimezone: true }).notNull(),
  effectiveDate: timestamp('effective_date', { withTimezone: true }).notNull(),
  createdBy: uuid('created_by').references(() => users.userId).notNull(),
  createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
}, (table) => [
  index('ima_amendments_type_idx').on(table.amendmentType),
  index('ima_amendments_effective_date_idx').on(table.effectiveDate),
]);
```

**ima_amendment_responses table:**
```typescript
export const imaAmendmentResponses = pgTable('ima_amendment_responses', {
  responseId: uuid('response_id').primaryKey().defaultRandom(),
  amendmentId: uuid('amendment_id').references(() => imaAmendments.amendmentId).notNull(),
  providerId: uuid('provider_id').references(() => providers.providerId).notNull(),
  responseType: varchar('response_type', { length: 20 }).notNull(),  // ACKNOWLEDGED, ACCEPTED, REJECTED
  respondedAt: timestamp('responded_at', { withTimezone: true }).defaultNow().notNull(),
  ipAddress: varchar('ip_address', { length: 45 }).notNull(),
  userAgent: varchar('user_agent', { length: 500 }).notNull(),
}, (table) => [
  index('ima_responses_amendment_idx').on(table.amendmentId),
  index('ima_responses_provider_idx').on(table.providerId),
  uniqueIndex('ima_responses_unique_idx').on(table.amendmentId, table.providerId),
]);
```

### 3. Update `packages/shared/src/schemas/db/index.ts`:

Export the two new tables.

## Critical Security Rules

- `documentHash` must be SHA-256 of the amendment document text, computed server-side. Never accept hash from the client.
- `ipAddress` and `userAgent` are captured from the HTTP request at acknowledgement/response time, not from client-submitted data.
- The unique index on (amendmentId, providerId) prevents duplicate responses.
